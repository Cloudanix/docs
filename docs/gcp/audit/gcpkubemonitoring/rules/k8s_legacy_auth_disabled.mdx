### More Info:

Ensure legacy authorization is set to disabled on Kubernetes clusters. The legacy authorizer in Kubernetes grants broad, statically defined permissions.

### Risk Level

Medium

### Address

Security

### Compliance Standards

SOC2, NISTCSF, PCIDSS



<Tabs><Tab title='Cause'>
### Check Cause

#### Using Console

1. Log in to your Google Cloud Platform (GCP) console.

2. Navigate to the Kubernetes Engine section. You can do this by clicking on the hamburger menu on the top left corner, then select "Kubernetes Engine".

3. In the Kubernetes Engine section, you will see a list of all your Kubernetes clusters. Click on the name of the cluster you want to inspect.

4. In the cluster details page, scroll down to the "Cluster Features" section. Here, you should see a feature called "Legacy Authorization". If this feature is enabled, it means that Legacy Authorization is not disabled in your Kubernetes cluster.

#### Using CLI

1. Install and configure Google Cloud SDK: Before you can use the GCP CLI, you need to install the Google Cloud SDK. You can download it from the official Google Cloud website. After downloading, install it and authenticate it with your Google Cloud account using the command `gcloud auth login`.

2. List all the clusters: Use the following command to list all the Kubernetes clusters in your GCP project:

   ```
   gcloud container clusters list
   ```

   This command will return a list of all the Kubernetes clusters along with their details.

3. Check the Legacy Authorization status: For each cluster, check the status of Legacy Authorization. You can do this using the following command:

   ```
   gcloud container clusters describe [CLUSTER_NAME] --zone [ZONE_NAME] --format='value(legacyAbac.enabled)'
   ```

   Replace `[CLUSTER_NAME]` with the name of your cluster and `[ZONE_NAME]` with the zone of your cluster. This command will return either `true` or `false`. If it returns `true`, Legacy Authorization is enabled. If it returns `false`, Legacy Authorization is disabled.

4. Repeat the process: Repeat step 3 for each cluster in your GCP project. If any cluster returns `true`, it means that Legacy Authorization is enabled for that cluster, which is a misconfiguration.

#### Using Python

1. Install the necessary Python libraries: You will need the `google-cloud-container` library to interact with Google Kubernetes Engine (GKE). You can install it using pip:

   ```bash
   pip install --upgrade google-cloud-container
   ```

2. Import the necessary modules and initialize the client:

   ```python
   from google.cloud import container_v1

   client = container_v1.ClusterManagerClient()
   ```

3. Get the list of all clusters in the project and check the legacy authorization status:

   ```python
   project_id = 'your-project-id'
   zone = 'your-zone'  # e.g., 'us-central1-a'

   response = client.list_clusters(project_id, zone)

   for cluster in response.clusters:
       print(f"Cluster name: {cluster.name}")
       print(f"Legacy authorization: {cluster.legacy_abac.enabled}")
   ```

4. Analyze the output: The script will print the name of each cluster and whether legacy authorization is enabled. If `legacy_abac.enabled` is `True` for any cluster, it means that legacy authorization is enabled for that cluster, which is a misconfiguration.

Please replace `'your-project-id'` and `'your-zone'` with your actual project ID and zone. Also, ensure that you have the necessary permissions to list and view the clusters in the project.

### Remediation

#### Using Console

To remediate the "Legacy Authorization Should Be Disabled" misconfiguration in GCP using GCP console, follow these steps:

1. Open the GCP console and navigate to the IAM & Admin page.
2. Click on the "Settings" tab.
3. Scroll down to the "Authorization" section.
4. Under "Legacy authorization", click on the "Edit" button.
5. Select the checkbox for "Disable project-wide legacy authorization".
6. Click on the "Save" button to apply the changes.

Once you have completed these steps, legacy authorization will be disabled for your GCP project. It is recommended to periodically review your IAM settings to ensure that they are up-to-date and secure.

#### Using CLI

To remediate the "Legacy Authorization Should Be Disabled" misconfiguration for GCP using GCP CLI, you can follow the below steps:

1. Open the Google Cloud Console and go to the Cloud Shell.

2. In the Cloud Shell, run the following command to check if legacy authorization is enabled:

   ```
   gcloud projects get-iam-policy <project-id>
   ```

   Replace `<project-id>` with your GCP project ID.

3. If the output shows that legacy authorization is enabled, run the following command to disable it:

   ```
   gcloud beta iam service-accounts disable-legacy-iam <service-account-email>
   ```

   Replace `<service-account-email>` with the email address of the service account that you want to disable legacy authorization for.

4. After running the above command, you can verify that legacy authorization has been disabled by running the following command:

   ```
   gcloud projects get-iam-policy <project-id>
   ```

   The output should show that legacy authorization is now disabled.

Note: You need to have the necessary permissions to disable legacy authorization for a service account.

#### Using Python

To remediate the "Legacy Authorization Should Be Disabled" misconfiguration in GCP using Python, you can follow these steps:

1. Install the Google Cloud IAM API client library for Python using pip:

```
pip install --upgrade google-cloud-iam
```

2. Create a Python script and import the necessary libraries:

```python
from google.oauth2 import service_account
from google.cloud import iam_v1
```

3. Set up authentication by creating a service account and downloading the JSON key file. Then, create a credentials object using the JSON key file:

```python
key_path = '/path/to/key.json'
creds = service_account.Credentials.from_service_account_file(key_path)
```

4. Create an instance of the IAM client:

```python
client = iam_v1.IAMClient(credentials=creds)
```

5. Get the current IAM policy for the project:

```python
project_id = 'your-project-id'
resource = f'projects/{project_id}'
policy = client.get_iam_policy(request={'resource': resource})
```

6. Check if the "allUsers" or "allAuthenticatedUsers" member exists in any of the IAM policy's bindings:

```python
for binding in policy.bindings:
    if 'allUsers' in binding.members or 'allAuthenticatedUsers' in binding.members:
        binding.members = [m for m in binding.members if m not in ['allUsers', 'allAuthenticatedUsers']]
```

7. Update the IAM policy with the modified bindings:

```python
policy = client.set_iam_policy(request={'resource': resource, 'policy': policy})
```

8. Print a message indicating that the remediation is complete:

```python
print('Legacy authorization has been disabled.')
```

9. Save the script and run it using the command:

```
python script.py
```

This will remediate the "Legacy Authorization Should Be Disabled" misconfiguration in GCP by removing the "allUsers" and "allAuthenticatedUsers" members from any IAM policy bindings that contain them.
