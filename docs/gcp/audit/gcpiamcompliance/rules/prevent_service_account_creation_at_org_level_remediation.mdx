#### Using Console

1. Sign in to the Google Cloud Console: Open your web browser, navigate to the Google Cloud Console (console.cloud.google.com), and sign in with your Google account credentials.

2. Select your organization: Once you're in the console, you'll see a drop-down menu at the top of the page, next to "Google Cloud Console". Click on this drop-down and select the organization you want to check.

3. Navigate to IAM & Admin: On the left-hand side of the console, you'll see a navigation menu. Scroll down until you find "IAM & Admin" and click on it. This will take you to the IAM & Admin dashboard.

4. Check for service account creation permissions: In the IAM & Admin dashboard, you'll see a list of all the roles and permissions in your organization. Look for roles that have the "iam.serviceAccounts.create" permission. If any role has this permission, it means that service account creation is not prevented for your Google Cloud Organization.

#### Using CLI

1. Install and configure the Google Cloud SDK: The Google Cloud SDK includes the gcloud command-line tool, which is necessary to interact with your Google Cloud resources. You can install it by following the instructions provided by Google Cloud.

2. Authenticate your account: Before you can start using the gcloud command-line tool, you need to authenticate your account. You can do this by running the following command in your terminal:

   ```
   gcloud auth login
   ```
   Follow the prompts to authenticate your account.

3. List all the IAM policies for your organization: You can list all the IAM policies for your organization by running the following command:

   ```
   gcloud organizations get-iam-policy ORGANIZATION_ID
   ```
   Replace `ORGANIZATION_ID` with your actual organization ID. This command will return a list of all the IAM policies for your organization.

4. Check for service account creation permissions: In the list of IAM policies, look for any policies that have the `roles/iam.serviceAccountCreator` role. This role allows a user to create service accounts. If any policies have this role, then service account creation is not prevented for your organization.

4. Search for the "Service Account Admin" role and click on it.

```
gcloud organizations get-iam-policy [ORGANIZATION_ID] --flatten="constraints[]" --format="table(constraints.constraint|constraints.listPolicy.booleanPolicy.enforced)"
```

Note: Replace [ORGANIZATION_ID] with your actual organization ID.

```
