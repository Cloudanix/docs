---
title: KMS Admin Roles Should Not Have CryptoKey Role
---

### More Info:

Ensure that no users have the KMS admin role and any one of the CryptoKey roles follows separation of duties, where no user have access to resources out of the scope of duty.

### Risk Level

High

### Address

Security

### Compliance Standards

CBP, CIS

### Remediation

How to disable KMS admin role or CryptoKey role for user.

#### Using GCP Console

1. Log in to the Google Cloud Platform (GCP) Console using your GCP account credentials.
2. Navigate to the IAM & Admin section by selecting "IAM & Admin" from the main menu.
3. In the IAM & Admin page, select "IAM" to manage user roles and permissions.
4. Locate the user account that has the KMS Admin role and any CryptoKey role assigned.
   (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "KMS Admin Roles Should Not Have CryptoKey Role" Policy.)
5. Click on the user account to view its assigned roles and permissions.
6. Identify the KMS Admin role and any CryptoKey roles assigned to the user.
7. To remove the KMS Admin role:
   - Click on the "Edit" button next to the user account.
   - Find the KMS Admin role in the user's assigned roles.
   - Click on the "X" icon or remove button next to the KMS Admin role to remove it.
8. To remove any CryptoKey roles:
   - Within the user's assigned roles, locate any CryptoKey roles.
   - Click on the "X" icon or remove button next to each CryptoKey role to remove them.
9. Review the remaining roles assigned to the user and ensure they are necessary and appropriate.
10. Click on the "Save" button to apply the changes to the user's roles and permissions.
