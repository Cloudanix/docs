---
title: KMS Cryptokeys Should Not Be Public
---

### More Info:

Ensure that Cloud KMS cryptokeys are not anonymously or publicly accessible.

### Risk Level

High

### Address

Security

### Compliance Standards

PCIDSS, GDPR, APRA, MAS, NIST, CIS

### Remediation

How to disable KMS CryptoKeys public access.

#### Using GCP Console

1. Log in to the Google Cloud Platform (GCP) Console using your GCP account credentials.
2. Navigate to the Cloud Key Management Service (KMS) section by selecting "Cloud KMS" from the main menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "KMS Cryptokeys Should Not Be Public" Policy.)
3. Select the key ring that contains the public Cryptokeys you want to remediate.
4. In the key ring details page, click on the name of the public Cryptokey you want to remediate.
5. In the Cryptokey details page, click on the "Permissions" tab.
6. Review the existing permissions and identify any unnecessary or public access entries.
7. Click on the "Edit" button next to the permissions.
8. In the permissions editor, locate the public access entries (such as "allUsers" or specific members with public access) and remove them.
9. Ensure that only authorized users or service accounts have the necessary permissions to access the Cryptokey.
10. Click on the "Save" button to apply the changes to the Cryptokey's permissions.
11. Repeat these steps for any other public Cryptokeys in the key ring that require remediation.
