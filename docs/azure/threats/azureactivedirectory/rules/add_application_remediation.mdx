
### Event Information

#### Meaning

- The "Add application" event in Azure Active Directory refers to the action of creating a new application registration within the Azure AD tenant.
- This event indicates that a new application has been added to the Azure AD tenant, which can be used to authenticate and authorize users and services to access Azure resources.
- The event typically includes information such as the application name, unique identifier (client ID), and other metadata associated with the application registration.

### Remediation

#### Using Console

- Example: If security is impacted when adding an application in Azure Active Directory, it could be due to misconfiguration of permissions or granting excessive privileges to the application. For instance, if an application is granted read and write access to sensitive user data without proper authentication and authorization controls, it can lead to unauthorized access and data breaches.

- Remediation steps using Azure console:
  1. Sign in to the Azure portal (https://portal.azure.com) with appropriate credentials.
  2. Navigate to the Azure Active Directory service.
  3. Select "App registrations" from the left-hand menu.
  4. Locate the application that needs remediation and click on it.
  5. Review the application's permissions and ensure they are aligned with the principle of least privilege.
  6. Remove any unnecessary or excessive permissions granted to the application.
  7. Enable multi-factor authentication (MFA) for the application, if applicable.
  8. Regularly review and audit the application's access and permissions to ensure ongoing security.

Note: The specific steps may vary slightly depending on the Azure portal version and layout. It is important to consult the official Azure documentation for the most up-to-date instructions.

#### Using CLI

1. Example of security impact: If an application is added to Azure Active Directory (Azure AD) without proper access controls and permissions, it can lead to unauthorized access to sensitive data and resources. This can result in data breaches, unauthorized modifications, or misuse of resources.

2. Remediation steps using Azure CLI:
   - Use the `az ad app create` command to create a new application in Azure AD with the necessary security configurations.
   - Set appropriate permissions and access controls for the application using the `az ad app permission` and `az ad app permission grant` commands.
   - Regularly review and update the application's permissions and access controls to ensure they align with the principle of least privilege.

Note: The specific CLI commands and parameters may vary depending on the exact requirements and configurations of the Azure AD application. It is recommended to refer to the Azure CLI documentation for detailed usage and examples.
