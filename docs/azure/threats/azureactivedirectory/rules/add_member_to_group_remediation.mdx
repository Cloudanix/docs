
### Event Information

#### Meaning

- The "Add member to group" event in Azure Active Directory refers to the action of adding a user or another group as a member to an existing group within the Azure AD.
- This event is typically triggered when an administrator or a user with appropriate permissions performs the action of adding a member to a group.
- The event provides visibility into the changes made to group memberships, allowing administrators to track and audit user access and permissions within the Azure AD environment.

### Remediation

#### Using Console

- Example: If security is impacted with the "Add member to group" action in Azure Active Directory, it could potentially lead to unauthorized access to sensitive resources if a user is added to a group without proper authorization or validation.

To remediate this issue in Azure Active Directory using the Azure portal, you can follow these step-by-step instructions:

1. Sign in to the Azure portal (https://portal.azure.com) using your Azure AD administrator account.
2. Navigate to the Azure Active Directory service by clicking on "Azure Active Directory" in the left-hand menu.
3. In the Azure Active Directory blade, select "Groups" from the left-hand menu.
4. Locate the group that you want to modify and click on its name to open the group details.
5. In the group details page, click on the "Members" tab.
6. Review the list of existing members and ensure that only authorized users are part of the group.
7. To add a new member, click on the "Add members" button.
8. In the "Add members" dialog, search for the user or users that you want to add to the group.
9. Select the user(s) from the search results and click on the "Select" button.
10. Review the selected users and click on the "Add" button to add them to the group.
11. After adding the member(s), review the group membership to ensure that the correct users have been added.
12. Repeat these steps for any other groups that require modification.

By following these steps, you can remediate the security impact of adding members to a group in Azure Active Directory by ensuring that only authorized users are added and unauthorized access is prevented.

#### Using CLI

1. Example of security impact: If an unauthorized user gains access to the Azure Active Directory (AAD) and adds themselves to a privileged group, they may gain elevated permissions and potentially compromise the security of the AAD and associated resources.

2. Remediation steps using Azure CLI:
   - Step 1: Identify the unauthorized member added to the group using the following command:
     ```
     az ad group member list --group <group-object-id>
     ```
   - Step 2: Remove the unauthorized member from the group using the following command:
     ```
     az ad group member remove --group <group-object-id> --member-id <member-object-id>
     ```
   - Step 3: Review and adjust the permissions and access controls for the group to prevent similar unauthorized additions in the future.

Note: Replace `<group-object-id>` with the object ID of the group and `<member-object-id>` with the object ID of the unauthorized member.
