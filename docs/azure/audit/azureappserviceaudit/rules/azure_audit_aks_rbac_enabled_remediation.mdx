#### Using CLI

To enable Role-Based Access Control (RBAC) within Azure Kubernetes Services using Azure CLI, follow these steps:

1. Open the Azure CLI and login to your Azure account.

2. Select the subscription that contains the AKS cluster using the following command:

   ```
   az account set --subscription <subscription_id>
   ```

3. Set the name of your AKS cluster using the following command:

   ```
   AKS_CLUSTER_NAME=<aks_cluster_name>
   ```

4. Enable RBAC in your AKS cluster using the following command:

   ```
   az aks update --name $AKS_CLUSTER_NAME --resource-group <resource_group_name> --enable-rbac
   ```

   Replace `<resource_group_name>` with the name of the resource group that contains the AKS cluster.

5. Verify that RBAC is enabled in your AKS cluster using the following command:

   ```
   az aks show --name $AKS_CLUSTER_NAME --resource-group <resource_group_name> --query 'enableRbac'
   ```

   If RBAC is enabled, the command will return `true`.
