
### Event Information

#### Meaning

- The UpdateFunctionConfiguration20150331v2 event in awsrealtime for Lambda refers to an event that is triggered when there is a configuration update made to a Lambda function in AWS.
- This event indicates that changes have been made to the function's configuration settings, such as its memory allocation, timeout duration, environment variables, or resource requirements.
- By monitoring this event, you can track and analyze any updates or modifications made to your Lambda functions, ensuring that they are properly configured and optimized for performance.

#### Example

- One possible way security can be impacted with the UpdateFunctionConfiguration20150331v2 event in AWS Lambda is if the event is triggered by an unauthorized user or if the event is used to modify sensitive configuration settings of the Lambda function.

Remediation:
1. Implement strict IAM policies to control access to the Lambda function and its configuration. Ensure that only authorized users or roles have permissions to modify the function configuration.
2. Enable AWS CloudTrail to monitor and log all API calls related to the Lambda function. This will help in identifying any unauthorized or suspicious activity.
3. Regularly review and audit the Lambda function's configuration settings to ensure they align with security best practices. Use AWS Config to track and manage the configuration changes.

Example CLI command to update IAM policy for a Lambda function:
```
aws lambda update-function-configuration --function-name <function-name> --statement-id <statement-id> --action <action> --principal <principal> --source-arn <source-arn>
```

Note: Replace `<function-name>`, `<statement-id>`, `<action>`, `<principal>`, and `<source-arn>` with the appropriate values for your environment.
