### More Info:

Ensure that AWS Config service is configured to include Global resources in order to have complete visibility over the configuration changes made within your AWS account. Global resources are not tied to a specific AWS region and can be used in all regions. Supported Global resource types are IAM users, groups, roles and customer managed policies.

### Remediation

How to enable global services for AWS Config

#### Using AWS Console

1. Sign in to the AWS Management Console using an account with administrative privileges.
2. Open the AWS Config console.
3. In the left navigation pane, click on "Settings" and then "Edit".
4. Check the checkbox name "Include global resources (e.g., AWS IAM resources)"
8. Once you have enabled the desired global services, click on the "Save" button to apply the changes.
