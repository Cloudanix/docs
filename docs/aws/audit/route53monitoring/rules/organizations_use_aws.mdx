### More Info:

Ensure that Amazon Organizations service is currently in use to gain central control over the use of AWS services across multiple AWS accounts (using Service Control Policies) in order to help you comply with the security and compliance policies within your company. AWS Organizations is an account management tool that enables you to centralize multiple AWS accounts into an organization that you create and administer. Amazon Organizations is available to all customers at no additional cost and has two main feature sets: Consolidated Billing features â€“ which provides basic management tools that you can use to centrally manage all the accounts (master and member accounts) within your organization. With this feature you can get a combined view of AWS charges incurred by all your accounts and also take advantage of pricing benefits from aggregated usage.

To prevent the misconfiguration of not using AWS Organizations in Route 53 using the AWS Management Console, follow these steps:

1. **Create an AWS Organization:**
   - Sign in to the AWS Management Console.
   - Navigate to the AWS Organizations console at [AWS Organizations](https://console.aws.amazon.com/organizations/).
   - Click on "Create organization" and follow the prompts to set up your organization.

2. **Enable All Features:**
   - Once your organization is created, ensure that all features are enabled.
   - In the AWS Organizations console, go to the "Settings" section.
   - Click on "Enable all features" if it is not already enabled. This allows you to use the full set of AWS Organizations features, including consolidated billing and policy-based management.

3. **Invite Accounts to Your Organization:**
   - In the AWS Organizations console, go to the "Accounts" section.
   - Click on "Add account" and choose "Invite account" to invite existing AWS accounts to join your organization.
   - Follow the prompts to send invitations to the accounts you want to manage under your organization.

4. **Use Service Control Policies (SCPs):**
   - In the AWS Organizations console, go to the "Policies" section.
   - Click on "Create policy" to create a Service Control Policy (SCP).
   - Define policies that restrict or allow specific actions for accounts within your organization, ensuring that Route 53 configurations adhere to your organizational policies.
   - Attach the SCP to the appropriate organizational units (OUs) or accounts.

By following these steps, you can ensure that AWS Organizations is used effectively to manage Route 53 configurations and other resources within your AWS environment.

<Accordion title='Using CLI'>
To ensure that AWS Organizations is used in Route 53, you can follow these steps using the AWS CLI:

1. **Create an AWS Organization:**
   Ensure you have an AWS Organization set up. If you don't have one, you can create it using the following command:
   ```sh
   aws organizations create-organization --feature-set ALL
   ```

2. **Enable Service Access for Route 53:**
   Enable trusted access for AWS Organizations to manage Route 53 domains. This allows AWS Organizations to manage Route 53 on your behalf.
   ```sh
   aws organizations enable-aws-service-access --service-principal route53.amazonaws.com
   ```

3. **Attach a Service Control Policy (SCP):**
   Create and attach a Service Control Policy (SCP) to your organization or specific organizational units (OUs) to enforce the use of Route 53 within the organization.
   ```sh
   aws organizations create-policy --content file://policy.json --description "Enforce Route 53 usage" --name "Route53Policy" --type SERVICE_CONTROL_POLICY
   ```

   Example `policy.json` content:
   ```json
   {
     "Version": "2012-10-17",
     "Statement": [
       {
         "Effect": "Allow",
         "Action": "route53:*",
         "Resource": "*"
       }
     ]
   }
   ```

4. **Attach the SCP to the Root or OU:**
   Attach the created SCP to the root of the organization or to specific OUs to ensure that the policy is enforced.
   ```sh
   aws organizations attach-policy --policy-id <policy-id> --target-id <root-id-or-ou-id>
   ```

Replace `<policy-id>` with the ID of the policy you created and `<root-id-or-ou-id>` with the ID of the root or organizational unit you want to attach the policy to.

By following these steps, you can ensure that AWS Organizations is used to manage Route 53, thereby preventing misconfigurations related to its usage.
</Accordion>

def check_organizations_used():
    client = boto3.client('route53')
    response = client.list_hosted_zones()
    for zone in response['HostedZones']:
        if 'Config' in zone and 'PrivateZone' in zone['Config']:
            if zone['Config']['PrivateZone'] == False:
                print(f"Public hosted zone detected: {zone['Name']}")
                return False
    return True

if __name__ == "__main__":
