### More Info:

There should be one Amazon KMS Customer Master Key (CMK) created in your AWS account for the web tier in order to protect data that transits your AWS web stack, have full control over data encryption/decryption process, and meet compliance requirements.

### Risk Level

Medium

### Address

Security

### Compliance Standards

CBP

### Remediation

How to create Amazon KMS Customer Master Key (CMK) in your AWS account for the web tier?

#### Using AWS Console

1. Open the AWS Management Console and log in to your account.
2. Navigate to the KMS console by selecting “KMS” from the list of services.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Web-tier KMS Key Should Be In Use" Policy.)
3. Click on “Create Key” to create a new KMS key.
4. Select “Symmetric” as the key type.
5. Select “AWS Key Management Service” as the key material origin.
6. Under “Key Administrators”, add the AWS account IDs of the users who will manage the key.
7. Under “Key Users”, add the AWS account IDs of the users who will use the key.
8. Click “Next” to proceed to the next screen.
9. Enter a name for your key in the “Alias” field.
10. Select a key policy that allows the web tier to use the key. You can either create a new policy or use an existing one.
11. Review your key configuration and click “Finish” to create the key.
