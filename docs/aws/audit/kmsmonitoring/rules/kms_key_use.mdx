### More Info:

You should have KMS CMK customer-managed keys in use in your account instead of AWS managed-keys in order to have full control over your data encryption and decryption process.

How to ensure that KMS CMK customer-managed keys in use in your account instead of AWS managed-keys?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the KMS service.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "KMS Customer Master Key Should Be In Use" Policy.)
2. Click on the "Customer managed keys" option in the left-hand navigation pane.
3. Verify that there are one or more customer-managed keys listed. If there are no customer-managed keys listed, then you are using AWS managed-keys.
4. To create a new customer-managed key, click on the "Create key" button.
5. Follow the prompts to create the key, and make sure to select "Customer managed key" as the key type.
6. Once the key is created, you can use it to encrypt and decrypt data in your AWS account.
7. To ensure that all new resources use your customer-managed key, you can update your AWS account's default KMS key policy.
8. Navigate to the KMS service and click on the "Customer managed keys" option in the left-hand navigation pane.
9. Click on the key for which you want to update the policy.
10. Click on the "Key policy" tab.
11. Click on the "Edit" button to edit the policy.
12. Update the policy to include your customer-managed key as the default key.
13. Click on the "Save changes" button to save the updated policy.
