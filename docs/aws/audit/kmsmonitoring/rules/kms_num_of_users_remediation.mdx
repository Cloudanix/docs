### Remediation

How to limit KMS admins?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the AWS Key Management Service (KMS).
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "KMS Key Policies Should Be Designed To Limit Number Of KMS Admins" Policy.)
2. From the left navigation pane, click on "Customer managed keys" to view the list of KMS keys.
3. Select the KMS key for which you want to limit admins by clicking on its name.
4. In the key details page, scroll down to the "Key administrators" section.
5. Review the list of key administrators and ensure that only authorized individuals or roles have administrative access to the key.
6. To limit the admins, click on the "Add" button in the "Key administrators" section.
7. In the "Add key administrators" dialog box, specify the IAM users or roles that should have administrative access to the key.
8. Click on the "Add" button to add the selected users or roles as key administrators.
9. If there are existing administrators who should no longer have administrative access, click on the "X" button next to their names to remove them.
10. Once you have updated the key administrators, click on the "Save changes" button to apply the changes.
11. Repeat the steps for each KMS key for which you want to limit the administrators.
