---
title: CloudTrail Must Log Data Events
---

### More Info:

Your AWS CloudTrail trails should be configured to log Data events in order to record S3 object-level API operations, such as GetObject, DeleteObject and PutObject.

### Risk Level

Low

### Address

Security

### Compliance Standards

SOC2, GDPR, PCIDSS, NIST, CIS, ISO27001, HIPAA

### Remediation

How to configure CloudTrail to log Data events?

#### Using AWS Console

1. Sign in to the AWS Management Console.
2. Navigate to the CloudTrail service.
   (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "CloudTrail Must Log Data Events" Policy.)
3. Click on "Trails" in the left navigation pane.
4. Click on the name of the existing trail you want to modify or click on "Create trail" to create a new trail.
5. In the trail configuration page, ensure that the trail is configured to log management events by selecting the appropriate S3 bucket, IAM role, and options.
6. Under "Event selector," click on "Add data event" to configure the data events you want to log.
7. In the "Data event settings" section, select the services for which you want to log data events. You can choose specific services or select "All supported services" to capture data events for all services.
8. For each selected service, choose the specific data events you want to log. You can select from a range of options such as S3 object-level events, Lambda function invocations, RDS database events, etc.
9. Click on "Save" to save the changes to the trail configuration.
10. If you created a new trail, ensure that it is enabled by clicking on the "Trails" tab and then clicking on the "Enable logging" button next to the trail name.
