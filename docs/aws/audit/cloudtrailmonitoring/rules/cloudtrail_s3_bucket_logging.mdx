---
title: Server Access Logging Feature Should Be Enabled
---

### More Info:

Any S3 buckets used by AWS CloudTrail should have Server Access Logging feature enabled in order to track requests for accessing the buckets and necessary for security audits.

### Risk Level

Medium

### Address

Security

### Compliance Standards

CIS

### Remediation

How to enable Server Access Logging for CloudTrail bucket?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the S3 service.
2. Find and select the S3 bucket where you want to enable Server Access Logging for CloudTrail.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Server Access Logging Feature Should Be Enabled" Policy.)
3. Click on the "Properties" tab in the bucket details view.
4. Scroll down to the "Server access logging" section and click on the "Edit" button.
5. In the "Server access logging" dialog, check the box next to "Enable logging" to enable logging for the bucket.
6. Select the target bucket where you want to store the access logs. This should be a separate bucket from your CloudTrail bucket.
7. Optionally, specify a prefix for the access log object keys in the "Target prefix" field.
8. Click the "Save" button to enable Server Access Logging for the bucket.
