### Remediation

How to enable encryption for CloudTrail logs

#### Using GCP Console

1. Open the AWS Management Console and navigate to the CloudTrail service.
2. In the CloudTrail console, select the specific trail for which you want to enable encryption or create a new trail if needed.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "CloudTrail Logs Should Be Encrypted" Policy.)
3. Click on the "Edit" button for the trail you selected.
4. In the "Data events" section, scroll down to the "Storage location" settings.
5. Enable the option "Enable log file encryption" by selecting the checkbox.
6. Choose the desired AWS Key Management Service (KMS) key from the drop-down list. You can either select an existing KMS key or create a new one.
7. Click on the "Save" button to save the changes.
8. Once the encryption is enabled, CloudTrail will encrypt the log files using the specified KMS key when storing them in the destination S3 bucket.
