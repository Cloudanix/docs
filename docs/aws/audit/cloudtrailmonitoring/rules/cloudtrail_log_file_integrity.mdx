---
title: File Integrity Validation Feature Should Be Enabled For Trails
---

### More Info:

Your trails should have file integrity validation feature enabled in order to check the log files and detect whether these were modified or deleted after CloudTrail agent delivered them to the S3 bucket.

### Risk Level

Medium

### Address

Security

### Compliance Standards

CIS, SOC2, PCIDSS, GDPR, NIST, ISO27001, HIPAA

### Remediation

How to enable file integrity validation for CloudTrail trails.

#### Using AWS Console

1. Open the AWS Management Console and navigate to the CloudTrail service.
2. Select the CloudTrail trail for which you want to enable file integrity validation.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "File Integrity Validation Feature Should Be Enabled For Trails" Policy.)
3. Click on the "Trails" menu on the left side, and then click on the name of the trail you want to configure.
4. In the trail details page, click on the "Edit" button.
5. Scroll down to the "Advanced" section.
6. In the "Log file integrity validation" section, toggle the switch to enable file integrity validation.
7. Choose the appropriate settings for log file integrity validation:
   - "SHA-256": Select this option to use SHA-256 hash algorithm for file integrity validation.
   - "SHA-256 and digest file": Select this option to use both SHA-256 hash algorithm and a separate digest file for file integrity validation. The digest file contains the cryptographic hashes for each CloudTrail log file.
8. Save the changes by clicking on the "Save" button.
