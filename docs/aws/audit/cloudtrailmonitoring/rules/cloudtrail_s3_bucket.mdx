### More Info:

Your Amazon CloudTrail trail should be configured to use the appropriated S3 bucket in order to meet regulatory compliance requirements within your organization

### Risk Level

Medium

### Address

Security

### Compliance Standards

SOC2

### Remediation

How to reconfigure Amazon CloudTrail trail to use the designated S3 bucket?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the CloudTrail service.
2. Click on the "Trails" menu on the left side.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "CloudTrail Should Be Configured To Use Appropriated S3 Bucket" Policy.)
3. Select the trail for which you want to configure the S3 bucket.
4. Click on the trail name to access the trail configuration.
5. In the trail configuration page, click on the "Edit" button.
6. Scroll down to the "Storage location" section.
7. Choose the desired S3 bucket from the "S3 bucket" dropdown menu. If the bucket you want to use is not listed, you can create a new bucket by clicking the "Create new bucket" button.
8. Optionally, you can specify a prefix for the log file names by entering it in the "S3 key prefix" field.
9. Save the changes by clicking on the "Save" button.
