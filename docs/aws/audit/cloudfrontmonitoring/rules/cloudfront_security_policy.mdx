---
title: CloudFront Distributions Should Use Security Policies With Appropriate Version And Ciphers
---

### More Info:

Your Amazon CloudFront distributions should use a security policy with minimum TLSv1.1 or TLSv1.2 and appropriate security ciphers for HTTPS viewer connections.

### Risk Level

Medium

### Address

Security

### Compliance Standards

PCIDSS, HIPAA, MAS, NIST

### Remediation

How to enable a security policy that enforce TLS V1.1 or V1?2 as the minimum protocol version for CloudFront distribution?

#### Using AWS Console

1. Open the CloudFront console:
   - Sign in to the AWS Management Console.
   - Go to the CloudFront service.
     (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "CloudFront Distributions Should Use Security Policies With Appropriate Version And Ciphers" Policy.)
2. Select the desired CloudFront distribution:
   - Locate and select the CloudFront distribution for which you want to enable the security policy.
3. Edit the distribution:
   - Click on the "Distribution Settings" tab.
   - Click on the "Edit" button to modify the distribution settings.
4. Configure SSL/TLS settings:
   - Scroll down to the "Security" section.
   - In the "Minimum SSL/TLS Version" option, select the desired minimum protocol version (TLSv1.1 or TLSv1.2).
   - Choose the appropriate protocol version based on your security requirements.
5. Save the changes:
   - Click on the "Save Changes" button to update the distribution settings.
