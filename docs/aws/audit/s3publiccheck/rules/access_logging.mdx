### More Info:

AWS S3 Server Access Logging feature should be enabled in order to record access requests useful for security audits. By default, server access logging is not enabled for S3 buckets.

### Remediation

How to enable AWS S3 Server Access Logging feature?

#### Using AWS Console

1. Sign in to the AWS Management Console.
2. Open the Amazon S3 console by navigating to the "Services" menu and selecting "S3" under the "Storage" section.
3. In the S3 console, select the bucket for which you want to enable server access logging.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "S3 Buckets Should Have Access Logging Enabled" Policy.)
4. Click on the "Properties" tab in the bucket details.
5. Under the "Advanced settings" section, click on "Server access logging".
6. Click on the "Edit" button to enable server access logging for the bucket.
7. In the "Server access logging" dialog, check the box next to "Enable logging".
8. Specify the target bucket where you want to store the access logs. You can choose an existing bucket or create a new one. Note that the target bucket must be in the same AWS region as the source bucket.
9. Optionally, you can specify a log prefix to organize the logs within the target bucket. The log prefix is a string that is added as a prefix to the generated log object key.
10. Click on the "Save changes" button to enable server access logging.
