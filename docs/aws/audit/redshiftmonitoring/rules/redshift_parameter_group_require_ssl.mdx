### More Info:

AWS Redshift non-default parameter groups require SSL to secure data in transit.

### Remediation

How to enable require_ssl parameter in Amazon Redshift non-default parameter groups?

#### Using AWS Console

1. Create a new parameter group or identify an existing non-default parameter group that you want to modify.
2. Navigate to the Amazon Redshift console and go to the "Parameter Groups" page.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Redshift Parameter Group Require SSL" Policy.)
3. Select the non-default parameter group you want to modify and click on the "Actions" dropdown menu.
4. Choose "Edit Parameters" to modify the parameters in the group.
5. In the parameter group editor, search for the `require_ssl` parameter. If it doesn't exist, you can add it by clicking on the "Add" button and entering the following details:
   - Name: `require_ssl`
   - Type: `boolean`
   - Value: `true`
6. Save the changes to the parameter group.
7. Associate the modified parameter group with your Redshift cluster. You can do this by selecting the cluster in the Redshift console, going to the "Configuration" tab, and choosing the modified parameter group in the "Parameter group" dropdown.
8. Apply the parameter group changes to the cluster by clicking on "Apply Changes" or by rebooting the cluster.
