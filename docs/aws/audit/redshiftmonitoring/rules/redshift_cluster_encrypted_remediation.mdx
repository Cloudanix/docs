To ensure that your Amazon Redshift clusters are encrypted using the AWS CLI, you can follow these steps:

1. **Create a KMS Key**:
   First, you need to create a KMS (Key Management Service) key that will be used to encrypt your Redshift clusters.
   ```sh
   aws kms create-key --description "Redshift encryption key"
   ```

2. **Create a Redshift Cluster with Encryption**:
   When creating a new Redshift cluster, specify the KMS key to enable encryption.
   ```sh
   aws redshift create-cluster \
       --cluster-identifier my-redshift-cluster \
       --node-type dc2.large \
       --master-username myuser \
       --master-user-password mypassword \
       --cluster-type single-node \
       --kms-key-id <KMS_KEY_ARN>
   ```

3. **Modify an Existing Redshift Cluster to Enable Encryption**:
   If you have an existing Redshift cluster, you need to take a snapshot, then restore the cluster from the snapshot with encryption enabled.
   ```sh
   aws redshift create-cluster-snapshot \
       --cluster-identifier my-redshift-cluster \
       --snapshot-identifier my-snapshot
   ```

   ```sh
   aws redshift restore-from-cluster-snapshot \
       --snapshot-identifier my-snapshot \
       --cluster-identifier my-new-redshift-cluster \
       --kms-key-id <KMS_KEY_ARN>
   ```

4. **Verify Encryption**:
   Finally, verify that the cluster is encrypted by describing the cluster and checking the `Encrypted` field.
   ```sh
   aws redshift describe-clusters \
       --cluster-identifier my-new-redshift-cluster \
       --query 'Clusters[*].{ClusterIdentifier:ClusterIdentifier,Encrypted:Encrypted}'
   ```

By following these steps, you can ensure that your Amazon Redshift clusters are encrypted, thereby preventing the misconfiguration of having unencrypted clusters.

   ```
   aws redshift describe-clusters --query 'Clusters[*].[ClusterIdentifier]' --output text

   ```
