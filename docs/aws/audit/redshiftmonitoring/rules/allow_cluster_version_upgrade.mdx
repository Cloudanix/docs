### More Info:

Version Upgrade should be enabled for Redshift clusters to automatically receive upgrades during the maintenance window.

### Risk Level

Low

### Address

Operational Maturity, Security

<Accordion title='Using CLI'>
To prevent Redshift Clusters from allowing version upgrades using the AWS CLI, you can follow these steps:

1. **Describe the Redshift Cluster:**
   First, you need to identify the current settings of your Redshift cluster. Use the following command to describe your Redshift cluster and check the `AllowVersionUpgrade` parameter.

   ```sh
   aws redshift describe-clusters --cluster-identifier <your-cluster-identifier>
   ```

2. **Modify the Redshift Cluster:**
   If the `AllowVersionUpgrade` parameter is set to `true`, you need to modify the cluster to set it to `false`. Use the following command to update the cluster configuration:

   ```sh
   aws redshift modify-cluster --cluster-identifier <your-cluster-identifier> --allow-version-upgrade false
   ```

3. **Verify the Changes:**
   After modifying the cluster, verify that the changes have been applied correctly by describing the cluster again.

   ```sh
   aws redshift describe-clusters --cluster-identifier <your-cluster-identifier>
   ```

4. **Automate the Check:**
   To ensure that this setting remains consistent, you can create a script or use AWS Config rules to continuously monitor and enforce this configuration. For example, you can use a simple bash script to check and update the setting periodically.

   ```sh
   #!/bin/bash
   CLUSTER_ID=<your-cluster-identifier>
   ALLOW_VERSION_UPGRADE=$(aws redshift describe-clusters --cluster-identifier $CLUSTER_ID --query 'Clusters[0].AllowVersionUpgrade' --output text)

   if [ "$ALLOW_VERSION_UPGRADE" == "true" ]; then
       aws redshift modify-cluster --cluster-identifier $CLUSTER_ID --allow-version-upgrade false
       echo "Updated AllowVersionUpgrade to false for cluster $CLUSTER_ID"
   else
       echo "AllowVersionUpgrade is already set to false for cluster $CLUSTER_ID"
   fi
   ```

By following these steps, you can prevent Redshift clusters from allowing version upgrades using the AWS CLI.
