---
title: Redshift Cluster Audit Should Have Logging Enabled
---

### More Info:

Audit logging should be enabled for Redshift clusters for security and troubleshooting purposes.

### Risk Level

Medium

### Address

Security

### Compliance Standards

PCIDSS, HIPAA, GDPR, APRA, MAS, NIST

### Remediation

How to determine if audit logging is enabled for your Amazon Redshift clusters?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the Amazon Redshift service.
2. Select the Redshift cluster for which you want to check the audit logging status.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Redshift Cluster Audit Should Have Logging Enabled" Policy.)
3. In the cluster details page, navigate to the "Properties" tab.
4. Look for the "Audit Logging" section.
5. If the audit logging is enabled, you will see information about the audit logging settings, such as the S3 bucket where the logs are stored, the prefix for log file names, and the IAM role used for access to the S3 bucket.
6. If the audit logging is not enabled, you will see a message indicating that audit logging is disabled.
