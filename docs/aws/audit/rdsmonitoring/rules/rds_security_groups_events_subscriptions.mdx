---
title: Security Groups Events Subscriptions Should Be Enabled
---

### More Info:

Amazon RDS event notification subscriptions should be enabled for database security groups events. AWS RDS groups these events into categories that you can subscribe to.

### Risk Level

Low

### Address

Reliability, Operational Maturity

### Compliance Standards

APRA, MAS, NIST

### Remediation

How to enable security groups events subscription?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the Amazon RDS service.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Security Groups Events Subscriptions Should Be Enabled" Policy.)
2. Click on the "Event subscriptions" link in the left navigation pane.
3. Click on the "Create event subscription" button.
4. Provide a name for the event subscription in the "Name" field.
5. Under the "Event categories" section, select the checkbox for "Security group change" to enable security group events.
6. Specify the SNS topic to which the event notifications will be sent. You can choose an existing topic or create a new one by clicking on the "Create new SNS topic" link.
7. Optionally, you can configure other settings such as the event severity, whether to include event metadata, and the event notification time window.
8. Review the event subscription configuration to ensure all the details are correct.
9. Click on the "Create" button to create the event subscription.
10. The event subscription will be created, and notifications will start being sent to the specified SNS topic when changes occur in the security groups associated with your RDS instances.
