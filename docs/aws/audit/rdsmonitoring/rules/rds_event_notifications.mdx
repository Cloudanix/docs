---
title: Event Notifications Should Be Enabled
---

### More Info:

Your AWS RDS resources should have event notifications enabled in order to be notified when an event occurs for a given database instance, database snapshot, database security group or database parameter group

### Risk Level

Low

### Address

Reliability, Operational Maturity

### Compliance Standards

NIST

### Remediation

How to enable event notification subscriptions for RDS

#### Using AWS Console

1. Open the AWS Management Console and navigate to the Amazon RDS service.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Event Notifications Should Be Enabled" Policy.)
2. Click on the "Event subscriptions" link in the left navigation pane.
3. Click on the "Create event subscription" button.
4. Provide a name for the event subscription in the "Name" field.
5. Specify the source type for the events. You can select "DB instance" to receive events related to a specific RDS instance or "DB cluster" to receive events related to an Aurora cluster.
6. Select the specific source identifier from the dropdown menu. This will be the name of the RDS instance or Aurora cluster for which you want to receive events.
7. Under the "Event categories" section, select the specific event categories for which you want to receive notifications. You can choose from a variety of categories, such as creation, deletion, modification, backup, failure, and more.
8. Specify the SNS topic to which the event notifications will be sent. You can choose an existing topic or create a new one by clicking on the "Create new SNS topic" link.
9. Optionally, you can configure other settings such as the event severity, whether to include event metadata, and the event notification time window.
10. Review the event subscription configuration to ensure all the details are correct.
11. Click on the "Create" button to create the event subscription.
12. The event subscription will be created, and notifications will start being sent to the specified SNS topic for the selected events.
