---
title: RDS Instance Should Be of Desired Type
---

### More Info:

It is recommended that RDS database instances use instance types from a limited set based on the database workload deployed.

### Risk Level

Medium

### Address

Reliability, Operational Maturity

### Compliance Standards

APRA, MAS

### Remediation

How to determine if RDS Instance is of Desired Type?

#### Using AWS Console

1. Sign in to the AWS Management Console.
2. Open the AWS Accounts and Billing console by navigating to the "Services" menu and selecting "AWS Accounts and Billing".
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "RDS Instance Should Be of Desired Type" Policy.)
3. In the AWS Accounts and Billing console, click on the "Reports" tab.
4. Under the "Create report" section, click on the "Create report" button.
5. In the "Create report" dialog box, select the report type as "Amazon RDS Usage".
6. Configure the report settings, including the report name, time range, and format.
7. Under the "Data to Include" section, select the desired data fields to be included in the report. You can choose fields like "DB Instance Identifier" and "DB Instance Class" to include information about the RDS instance type.
8. Click on the "Create report" button to generate the report.
9. Once the report is generated, you can download it in the specified format (e.g., CSV, Parquet) or view it directly in the AWS Accounts and Billing console.
10. Open the report and search for the specific RDS instance of interest. Look for the "DB Instance Class" column to determine if the instance is of the desired type.
