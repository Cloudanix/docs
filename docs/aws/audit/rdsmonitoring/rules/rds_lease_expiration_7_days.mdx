title: Renew RDS Reserved Instances Before Expiration (7 days)
---

### More Info:

Your AWS RDS Reserved Instances (RIs) should be renewed before expiration in order to get the appropriate discount on the hourly charge for these instances

### Risk Level

High

### Address

Cost Optimisation

### Compliance Standards

CBP

### Remediation

How to renew RDS reserved instances before expiration (7 days)?

#### Using AWS Console

1. Log in to the AWS Management Console using your AWS account credentials.
2. Navigate to the Amazon RDS service by selecting "RDS" from the services menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Renew RDS Reserved Instances Before Expiration (7 days)" Policy.)
3. In the RDS dashboard, click on "Reserved Instances" in the left navigation pane.
4. In the "Reserved Instances" page, you will see a list of your active reserved instances.
5. Identify the RDS database instance reservation(s) that are approaching expiration and need to be renewed.
6. Select the checkbox next to the reserved instance(s) you want to renew.
7. Above the list of reserved instances, click on the "Actions" button.
8. In the dropdown menu, select "Renew".
9. A Renew Reserved Instances window will appear, displaying the selected reservation details.
10. Review the reservation details and ensure they are correct.
11. Select the desired renewal term from the dropdown menu. Options may include 1 year or 3 years.
12. Choose whether you want to apply the renewal to the same instance type or exchange it for a different instance type, if available.
13. Click on the "Renew Reserved Instances" button to initiate the renewal process.
14. AWS will automatically handle the renewal and billing process based on your selected renewal term.
15. Once the renewal is complete, the reserved instance(s) will be extended with the new lease duration.
16. Monitor the expiration dates of your reserved instances and proactively repeat these steps to renew them as needed.
