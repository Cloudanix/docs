---
title: RDS Instances Should Make Use of Copy Tags
---

### More Info:

RDS instances should make use of Copy Tags to Snapshots feature in order to allow tags set on database instances to be automatically copied to any automated or manual RDS snapshots that are created from these instances

### Risk Level

Low

### Address

Cost Optimisation

### Compliance Standards

APRA, MAS

### Remediation

How to enable Copy Tags to Snapshot feature for RDS

#### Using AWS Console

1. Open the AWS Management Console and navigate to the Amazon RDS service.
2. Click on the name of the RDS instance for which you want to enable the "Copy Tags to Snapshot" feature.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "RDS Instances Should Make Use of Copy Tags" Policy.)
3. In the instance details page, click on the "Configuration" tab.
4. Under the "Database options" section, locate the "Additional configuration" option.
5. Click on the "Edit" button next to the "Additional configuration" option.
6. In the "Modify DB instance" window, locate the "Copy Tags to Snapshots" setting.
7. Set the "Copy Tags to Snapshots" setting to "Yes" to enable the feature.
8. Click on the "Apply immediately" checkbox to apply the changes immediately. If you don't select this option, the changes will be applied during the next maintenance window for the RDS instance.
9. Click on the "Modify DB instance" button to save the changes and enable the "Copy Tags to Snapshot" feature.
10. The "Copy Tags to Snapshot" feature will now be enabled for the selected RDS instance. Tags assigned to the RDS instance will be automatically copied to its snapshots.
