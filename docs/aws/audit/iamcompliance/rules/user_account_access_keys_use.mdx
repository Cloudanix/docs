### More Info:

Inactive access keys should be dropped.

### Risk Level

Medium

### Address

Security

### Compliance Standards

HIPAA, GDPR, NIST, SOC2, CISAWS, CBP

### Remediation

How to remove any unused IAM users access keys from your AWS account?

#### Using AWS Console

1. Sign in to the AWS Management Console using an account with administrative privileges.
2. Open the IAM console by navigating to the IAM service.
3. In the navigation pane, click on "Users" to view the list of IAM users in your account.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Inactive User Account Access keys Should Be Dropped" Policy.)
4. Locate the IAM user for whom you want to remove unused access keys.
5. Select the checkbox next to the IAM user's name to activate the actions menu.
6. Click on the "Security credentials" tab in the user details view.
7. In the "Access keys" section, you will see the list of access keys associated with the user.
8. Identify any access keys that are no longer in use and click on the "Make inactive" link next to each unused key. This will deactivate the access key.
9. Once you have deactivated all the unused access keys, select them by marking the checkboxes next to their names.
10. Click on the "Delete" button to permanently remove the selected access keys from your AWS account.
11. Confirm the action when prompted to delete the access keys.
12. Repeat these steps for any other IAM users in your account that have unused access keys.
