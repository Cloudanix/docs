title: Inactive Roles Should Be Removed
---

### More Info:

Inactive roles should be cleaned up.

### Risk Level

Medium

### Address

Security

### Compliance Standards

CBP

### Remediation

How to remove inactive roles?

#### Using AWS Console

1. Identify Inactive Roles:
   - Access the AWS Management Console and navigate to the IAM service.
   - Go to the "Roles" section and review the list of existing roles.
     (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Inactive Roles Should Be Removed" Policy.)
   - Identify the roles that are no longer in use or considered inactive.
2. Review Role Usage:
   - Examine Role Last Used Data:
     - Select each role and review the "Last Used" information in the "Permissions" tab.
     - Check when the role was last used to determine if it is active or inactive.
   - Analyze Trust Relationships:
     - Review the trust relationships of each role.
     - Ensure that the roles are not being used as trusted entities for other resources or services.
3. Document and Validate:
   - Verify Role Purpose:
     - Validate the purpose and necessity of each role.
     - Ensure that there are no pending or unfinished tasks associated with the roles.
   - Document Role Details:
     - Document the details of each role, including its purpose, owner, and any relevant information for future reference.
4. Remove Inactive Roles:
   - Access Role Deletion:
     - Select each inactive role that you want to remove.
     - Ensure that you have the necessary permissions to delete roles.
   - Delete the Roles:
     - Click on the "Delete role" button or navigate to the "Permissions" tab and select "Delete role" from the "Role actions" dropdown.
     - Confirm the deletion when prompted.
