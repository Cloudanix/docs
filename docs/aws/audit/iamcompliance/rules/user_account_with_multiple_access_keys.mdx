### More Info:

Multiple access keys for the same user should be avoided. There should be just 1 access key per user account.

### Risk Level

Medium

### Address

Security

### Compliance Standards

CISAWS, CBP, HITRUST

### Remediation

How to deactivate any unnecessary Amazon IAM access keys for a user?

#### Using AWS Console

1. Sign in to the AWS Management Console using an account with administrative privileges.
2. Open the IAM console by navigating to the IAM service.
3. In the navigation pane, click on "Users" to view the list of IAM users in your account.
4. Identify the IAM user for whom you want to deactivate unnecessary access keys.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "User Account Should Not Have Multiple Access Keys" Policy.)
5. Select the checkbox next to the IAM user's name to activate the actions menu.
6. Click on the "Security credentials" tab in the user details view.
7. Under the "Access keys" section, you will see a list of access keys associated with the user.
8. Identify the access keys that are unnecessary or no longer in use.
9. To deactivate an access key, select the access key and click on "Deactivate".
10. You will be prompted to confirm the deactivation. Click on "Yes, deactivate" to proceed.
11. The access key will be marked as inactive and can no longer be used for authentication.
12. Repeat the process for any other unnecessary access keys associated with the user.
