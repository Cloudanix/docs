### More Info:

Any unused IAM user without console access and API access should be removed as an extra security measure for protecting your AWS resources against unapproved access.

### Risk Level

Medium

### Address

Security

### Compliance Standards

CISAWS, CBP, HIPAA, SOC2, ISO27001, HITRUST, NISTCSF, PCIDSS

### Remediation

How to remove unused IAM accounts from AWS account?

### Remediation

How to remove unused IAM user

#### Using AWS Console

1. Sign in to the AWS Management Console using an account with administrative privileges.
2. Open the IAM console by navigating to the IAM service.
3. In the navigation pane, click on "Users" to view the list of IAM users in your account.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "User Account Without Any Usage Should Be Removed" Policy.)
4. Locate the unused IAM user that you want to remove from the list of users.
5. Select the checkbox next to the IAM user's name to activate the actions menu.
6. Click on the "Delete user" button in the actions menu.
7. AWS will display a confirmation message to ensure you want to delete the user. Review the details and ensure you are removing the correct user.
8. If you want to remove all access keys and console passwords associated with the user, select the checkboxes for "Delete login profile" and "Delete access keys".
9. Click on the "Delete" button to confirm the deletion of the unused IAM user.
10. AWS will delete the IAM user and associated resources, such as access keys, login profile, and attached policies.
