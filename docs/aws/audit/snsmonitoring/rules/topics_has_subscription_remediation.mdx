<Accordion title='Using CLI'>
To prevent the misconfiguration of SNS Topics without subscriptions using AWS CLI, you can follow these steps:

1. **List Existing SNS Topics**:
   First, you need to list all the existing SNS topics to identify which ones do not have subscriptions.
   ```sh
   aws sns list-topics
   ```

2. **Check Subscriptions for Each Topic**:
   For each SNS topic, check if there are any subscriptions associated with it.
   ```sh
   aws sns list-subscriptions-by-topic --topic-arn <topic-arn>
   ```

3. **Create a Subscription for the Topic**:
   If a topic does not have any subscriptions, create a subscription for it. You can subscribe an endpoint (e.g., an email address) to the topic.
   ```sh
   aws sns subscribe --topic-arn <topic-arn> --protocol email --notification-endpoint <email-address>
   ```

4. **Automate the Check and Subscription Process**:
   To ensure that all new topics have subscriptions, you can create a script that periodically checks for topics without subscriptions and automatically subscribes an endpoint to them.

   Example script:
   ```sh
   #!/bin/bash

   # List all SNS topics
   topics=$(aws sns list-topics --query 'Topics[*].TopicArn' --output text)

   for topic in $topics; do
       # Check subscriptions for each topic
       subscriptions=$(aws sns list-subscriptions-by-topic --topic-arn $topic --query 'Subscriptions[*].SubscriptionArn' --output text)
       
       if [ -z "$subscriptions" ]; then
           # If no subscriptions, create one
           aws sns subscribe --topic-arn $topic --protocol email --notification-endpoint <email-address>
           echo "Subscribed <email-address> to $topic"
       fi
   done
   ```

By following these steps, you can ensure that all SNS topics have at least one subscription, thereby preventing the misconfiguration.

1. Sign in to the AWS Management Console.
2. Navigate to the Simple Notification Service (SNS) dashboard by selecting "Services" from the top menu, then typing "SNS" in the search box and selecting "Simple Notification Service".

<Accordion title='Using CLI'>
1. First, you need to install and configure AWS CLI on your local machine. You can do this by following the instructions provided by AWS. Make sure you have the necessary permissions to access the SNS topics.

2. Once the AWS CLI is installed and configured, you can list all the SNS topics in your AWS account by running the following command:

   ```
   aws sns list-topics
   ```
   This command will return a list of all SNS topics in your AWS account.

3. For each SNS topic, you can list all the subscriptions by running the following command:

   ```
   aws sns list-subscriptions-by-topic --topic-arn <TopicARN>
   ```
   Replace `<TopicARN>` with the ARN of the SNS topic. This command will return a list of all subscriptions for the specified SNS topic.

4. If the command in step 3 returns an empty list, it means that the SNS topic does not have any subscriptions. This is a misconfiguration because SNS topics should have at least one subscription to ensure that the messages published to the topic are delivered to the subscribers.
