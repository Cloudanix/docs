2. **Access EventBuses:**
   - In the CloudWatch dashboard, on the left-hand side, click on "Rules" under the "Events" section.
   - Then, click on "Event buses" to view your custom EventBuses.

3. **Review and Edit Resource Policies:**
   - Select the custom EventBus you want to manage.
   - Click on the "Resource-based policy" tab to view the current policies attached to the EventBus.
   - Carefully review the existing policies to ensure they follow the principle of least privilege.

4. **Restrict Policy Attachments:**
   - To prevent unauthorized policy attachments, ensure that only trusted IAM roles and users have permissions to modify the EventBus policies.
   - Go to the IAM service in the AWS Management Console.
   - Review and update IAM policies and roles to restrict permissions for actions like `events:PutPermission` and `events:RemovePermission` to only trusted entities.

By following these steps, you can help prevent unauthorized resource policy attachments to your custom EventBus in CloudWatch.

<Accordion title='Using CLI'>
To prevent Resource Policy Attachment in a Custom EventBus in CloudWatch using AWS CLI, you can follow these steps:

1. **Create a Custom EventBus:**
   Ensure you have a custom EventBus created. If not, create one using the following command:
   ```sh
   aws events create-event-bus --name my-custom-event-bus
   ```

2. **Attach a Resource Policy with Least Privilege:**
   Attach a resource policy to the EventBus that follows the principle of least privilege. This means only allowing specific actions from specific principals. For example:
   ```sh
   aws events put-permission --event-bus-name my-custom-event-bus --action "events:PutEvents" --principal "123456789012" --statement-id "AllowPutEventsFromSpecificAccount"
   ```

3. **Review and List Attached Policies:**
   Regularly review the policies attached to your EventBus to ensure no unnecessary permissions are granted. Use the following command to list the policies:
   ```sh
   aws events list-permissions --event-bus-name my-custom-event-bus
   ```

4. **Remove Unnecessary Policies:**
   If you find any policies that should not be attached, remove them immediately. For example, to remove a specific policy:
   ```sh
   aws events remove-permission --event-bus-name my-custom-event-bus --statement-id "AllowPutEventsFromSpecificAccount"
   ```

By following these steps, you can ensure that only the necessary permissions are attached to your Custom EventBus, thereby preventing misconfigurations related to resource policy attachments.
</Accordion>

       "Statement": [

   policy_json = json.dumps(secure_policy)
