title: Internet Gateway Changes Alarm
---

### More Info:

AWS VPC Customer/Internet Gateway configuration changes should be monitored using CloudWatch alarms.

### Risk Level

Medium

### Address

Security

### Compliance Standards

CIS, SOC2, NIST, ISO27001, HIPAA

### Remediation

How to enable internet gateway changes alarm in Cloudwatch

#### Using GCP Console

1. Open the AWS Management Console and navigate to the CloudWatch service.
2. In the CloudWatch console, click on "Alarms" in the left navigation pane.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Internet Gateway Changes Alarm" Policy.)
3. Click on the "Create alarm" button.
4. In the "Create Alarm" wizard, select the "Select metric" button.
5. In the "Browse" tab, navigate to the "EC2" namespace.
6. Under "Per-Internet Gateway Metrics," select the "Status" metric.
7. Choose the specific Internet Gateway for which you want to monitor changes and click on the checkbox next to it.
8. Click on the "Select metric" button to proceed.
9. In the "Create Alarm" wizard, configure the alarm threshold and conditions based on your requirements. For example, you can set the threshold to ">= 1" to trigger an alarm if any change occurs.
10. Specify the actions to perform when the alarm state is triggered, such as sending a notification or executing an AWS Lambda function.
11. Provide a name and description for the alarm.
12. Click on the "Create alarm" button to create the alarm.
