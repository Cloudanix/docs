### More Info:

Identify and remove any unassociated Elastic IP (EIP) addresses for cost optimization.

### Risk Level

Low

### Address

Cost Optimisation

### Compliance Standards

AWSWAF, HITRUST, SOC2, NISTCSF

### Remediation

How to remediate unassociated Elastic IP addresses?

#### Using AWS Console

1. Access the AWS Management Console and navigate to the Amazon EC2 service.
2. From the left navigation pane, click on "Elastic IPs" under the "Network & Security" section.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unassociated Elastic IP Addresses Should Be Removed" Policy.)
3. Review the list of existing Elastic IP addresses.
4. Identify any Elastic IP addresses that are not associated with any EC2 instance or resource.
5. Select each unassociated Elastic IP address.
6. Click on the "Actions" button and choose "Release addresses" to release the selected Elastic IP addresses.
7. Confirm the release by clicking on the "Release" button in the confirmation dialog.
8. Repeat steps 5-7 for all unassociated Elastic IP addresses until none of them are remaining.
