### Remediation

How to update security group inbound configuration in order to restrict ICMP access to specific entities?

#### Using AWS Console

1. Log in to your AWS console and go to the EC2 dashboard.
2. In the navigation pane, click on "Security Groups".
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted ICMP Access Should Not Be Allowed" Policy.)
3. Select the security group that you want to modify.
4. In the "Inbound Rules" tab, locate the rule that allows ICMP traffic.
5. Click on the "Edit" button next to the rule.
6. In the "Source" field, select "Custom" from the dropdown menu.
7. Enter the IP address or CIDR block that you want to allow ICMP traffic from.
8. Click on the "Save" button to apply the changes.
9. Verify that the changes have been applied by testing the ICMP connectivity from the allowed entities.
