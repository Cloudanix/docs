### More Info:

No EC2 security group should allow unrestricted inbound access to any uncommon ports.

### Risk Level

Medium

### Address

Security

### Compliance Standards

SOC2, NIST, PCIDSS, AWSWAF

### Remediation

How to update EC2 security group inbound configuration to restrict access to specific entities on uncommon ports?

#### Using AWS Console

1. Log in to the AWS Management Console and navigate to the EC2 dashboard.
2. From the left-hand menu, select "Security Groups".
3. Select the security group that you want to modify and click on the "Inbound Rules" tab.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted Inbound Access on All Uncommon Ports Should Not Be Allowed" Policy.)
4. Locate the rule that allows access to uncommon ports and click the "Edit" button.
5. In the "Source" field, enter the IP address or IP range that should be allowed access to the port.
6. In the "Port Range" field, enter the specific port number or range of port numbers that should be allowed access.
7. Select the protocol (TCP, UDP, or ICMP) that should be allowed access to the port.
8. Click the "Save" button to apply the changes.
9. Verify that the changes have taken effect by testing the connection to the port from the specified IP address or range.
