### Remediation

How to update security group inbound configuration to restrict RDP port 3389 access to specific entities?

#### Using AWS Console

1. Go to the AWS Management Console and navigate to the EC2 dashboard.
2. Select the appropriate security group for which you want to restrict RDP access.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted RDP Access Should Not Be Allowed" Policy.)
3. Click on the "Inbound Rules" tab in the bottom panel.
4. Find the rule that allows RDP access on port 3389.
5. Click on the "Edit" button next to the rule.
6. In the "Source" field, select "Custom" and enter the IP address or range of IP addresses that you want to allow RDP access for.
7. Click on the "Save" button to apply the changes.
