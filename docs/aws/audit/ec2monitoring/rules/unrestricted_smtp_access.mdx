---
title: Unrestricted SMTP Access Should Not Be Allowed
---

### More Info:

No security group should allow unrestricted inbound access to TCP port 25 (SMTP).

### Risk Level

Medium

### Address

Security

### Compliance Standards

PCIDSS, APRA, MAS, NIST

### Remediation

How to update your security group inbound configuration to restrict SMTP TCP port 25 access to specific entities?

#### Using AWS Console

1. Log in to your AWS Management Console.
2. Navigate to the EC2 service.
3. Click on the "Security Groups" option in the left-hand menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted SMTP Access Should Not Be Allowed" Policy.)
4. Select the security group that needs to be updated.
5. Click on the "Inbound Rules" tab.
6. Locate the existing rule that allows TCP port 25 access.
7. Click on the "Edit" button next to the rule.
8. Change the source from "0.0.0.0/0" or "::/0" to the specific entities that should be allowed access. This can be done by specifying the IP address range or security group ID in the "Source" field.
9. Click on the "Save" button to apply the changes.
