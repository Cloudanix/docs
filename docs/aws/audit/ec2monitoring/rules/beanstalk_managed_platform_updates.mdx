### More Info:

Ensure that all your Amazon Elastic Beanstalk (EB) application environments have platform updates enabled in order to receive bug fixes, software updates and new features. Managed platform updates perform immutable environment updates.

### Risk Level

Medium

### Address

Operational Maturity, Reliability, Security

### Compliance Standards

ISO27001, HIPAA

### Remediation

How to enable managed platform updates for Beanstalk environments?

#### Using AWS Console

1. Log in to the AWS Management Console and navigate to the Elastic Beanstalk service.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Ensure Managed Platform Updates Are Enabled For Elastic Beanstalk Environment" Policy.)
2. Select the environment for which you want to enable managed platform updates.
3. Click on the "Configuration" tab in the left-hand menu.
4. Scroll down to the "Managed Updates" section and click on "Edit".
5. Select the "Enable managed platform updates" checkbox.
6. Choose the maintenance window during which you want the updates to be applied.
7. Click on "Apply" to save the changes.
