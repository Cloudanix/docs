### Remediation

How to ensure that no EC2 instances are launched in your AWS account using blocklisted instance types?

#### Using AWS Console

1. Determine the blacklisted instance types that you want to restrict within your AWS environment. This list should include the instance types that are not approved or allowed for use.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "EC2 Instances Should Not Have Blacklisted Instance Types" Policy.)
2. Log in to the AWS Management Console using your AWS account credentials.
3. Open the AWS Support Center by navigating to the AWS Support Console at https://console.aws.amazon.com/support/home.
4. Click on the "Create case" button to open a new support case.
5. In the "Create case" form, select "Account and Billing Support" as the category.
6. Select the appropriate severity level for your case, based on the urgency of the issue.
7. In the "Case description" field, provide a clear and concise description of the issue, indicating that you want to enforce the restriction of blacklisted instance types in your AWS environment.
8. Specify the list of blacklisted instance types in the case description, ensuring that it is comprehensive and up to date.
9. Submit the case to AWS Support.
10. AWS Support will review your case and provide guidance on the best approach to enforce the restriction of blacklisted instance types.
11. Follow the instructions provided by AWS Support, which may include:
    - Utilizing AWS Config rules to check for and remediate instances with blacklisted instance types.
    - Implementing custom scripts or automation tools to monitor and enforce the restriction.
    - Leveraging AWS Organizations or AWS Service Control Policies (SCPs) to enforce restrictions across multiple accounts.
12. Implement the recommended solution or approach provided by AWS Support.
