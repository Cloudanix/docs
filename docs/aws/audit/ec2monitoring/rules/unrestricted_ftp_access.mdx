---
title: Unrestricted FTP Access Should Not Be Allowed
---

### More Info:

No security group should allow unrestricted inbound access to TCP ports 20 and 21 (FTP).

### Risk Level

Medium

### Address

Security

### Compliance Standards

PCIDSS, APRA, MAS, NIST

### Remediation

How to update the security group inbound configuration to restrict FTP ports 21 and 22 access to specific entities?

#### Using AWS Console

1. Login to the AWS console and navigate to the EC2 dashboard.
2. Select the Security Group that needs to be updated.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted FTP Access Should Not Be Allowed" Policy.)
3. Click on the "Inbound rules" tab.
4. Locate the rule that allows FTP traffic on ports 21 and 22.
5. Click on the "Edit" button for that rule.
6. In the "Source" field, select "Custom" and enter the IP address or IP range of the entities that should have access to FTP traffic.
7. Click on the "Save rules" button to apply the changes.
