### Remediation

How to update the security group inbound configuration to restrict Elasticsearch TCP port 9200 access to specific entities?

#### Using AWS Console

1. Log in to the AWS Management Console.
2. Navigate to the EC2 service.
3. Click on the "Security Groups" option in the left-hand menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted Elasticsearch Access Should Not Be Allowed" Policy.)
4. Select the security group that is associated with the Elasticsearch instance.
5. Click on the "Inbound Rules" tab.
6. Scroll down to the rule that allows access to TCP port 9200.
7. Click on the "Edit" button for that rule.
8. Change the source IP address or range to the specific entities that should have access to Elasticsearch.
9. Click on the "Save" button to apply the changes.
10. Verify that the new rule is working as expected by testing access to Elasticsearch from the specified entities.
