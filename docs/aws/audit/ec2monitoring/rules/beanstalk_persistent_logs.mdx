---
title: Ensure Persistent Logs Are Enabled For Elastic Beanstalk Environments
---

### More Info:

Ensure that AWS Elastic Beanstalk (EB) environment logs are retained and uploaded to Amazon S3 in order to keep the logging data for future audits, historical purposes or to track and analyze the EB application environment behavior for a long period of time.

### Risk Level

Medium

### Address

Operational Maturity, Reliability

### Compliance Standards

NIST, GDPR

### Remediation

How to enable persistent logs for Beanstalk environments.

#### Using AWS Console

1. Log in to your AWS Management Console and navigate to the Elastic Beanstalk service dashboard.
2. Select the environment for which you want to enable persistent logs.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Ensure Persistent Logs Are Enabled For Elastic Beanstalk Environments" Policy.)
3. In the left-hand menu, click on "Configuration".
4. Scroll down to the "Software" section and click on "Edit".
5. In the "log streaming" section, select "Enable log file rotation" and "Enable log streaming".
6. Configure the log retention period and the log group name.
7. Click on "Apply" to save the changes.
8. Wait for a few minutes for the changes to take effect.
