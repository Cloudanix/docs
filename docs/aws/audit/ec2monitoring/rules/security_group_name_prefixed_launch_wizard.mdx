---
title: Security Group Name Prefixed With launch-wizard Should Not Be Used
---

### More Info:

EC2 security groups prefixed with launch-wizard should not be in use in order to follow AWS security best practices.

### Risk Level

Low

### Address

Security

### Compliance Standards

APRA, MAS

### Remediation

How to ensure security group name prefixed with 'launch-wizard' is not used

#### Using AWS Console

1. Access the AWS Management Console and navigate to the Amazon EC2 service.
2. From the left navigation pane, click on "Security Groups" under the "Network & Security" section.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Security Group Name Prefixed With launch-wizard Should Not Be Used" Policy.)
3. Review the list of existing security groups and identify any groups that have names starting with 'launch-wizard'.
4. Select each security group that has a name prefixed with 'launch-wizard'.
5. Click on the "Actions" button and choose "Delete security group" to remove the selected security groups.
6. Confirm the deletion by clicking on the "Yes, Delete" button in the confirmation dialog.
7. Repeat steps 4-6 for all security groups with names starting with 'launch-wizard' until none of them are remaining.
