### More Info:

EC2 instance launched should be from an approved list of instance types.

### Risk Level

Medium

### Address

Operational Maturity, Reliability, Security

### Compliance Standards

CBP

### Remediation

How to limit the EC2 instances that will be launched in your account to the desired instance type?

#### Using AWS Console

1. Log in to your AWS Management Console and navigate to the AWS Support Center.
2. Click on the "Create a Case" button.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "EC2 Instance Should Be of Desired Type" Policy.)
3. In the "Create a Case" form, select "Service Limit Increase" as the category.
4. In the "Limit Type" dropdown, select "EC2 Instances".
5. In the "Limit Name" dropdown, select the instance type that you want to limit.
6. In the "New Limit Value" field, enter the maximum number of instances of the selected type that you want to allow.
7. In the "Case Description" field, provide a brief description of the request.
8. Click "Submit" to create the case.
9. AWS Support will review your request and respond with either an approval or a request for more information.
10. If your request is approved, the limit on the number of instances of the selected type will be updated to the value that you specified.
