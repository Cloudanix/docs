### Remediation

How to update security groups inbound configuration to restrict HTTP access to specific entities?

#### Using AWS Console

1. Access the AWS Management Console and navigate to the Amazon EC2 service.
2. From the left navigation pane, click on "Security Groups" under the "Network & Security" section.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted HTTP Access Should Not Be Allowed" Policy.)
3. Review the list of existing security groups and identify the security group that you want to modify.
4. Select the desired security group.
5. In the "Inbound rules" tab, review the existing inbound rules.
6. Identify the rule that allows HTTP access, typically using the "HTTP" or "80" port.
7. Select the HTTP rule that you want to restrict to specific entities.
8. Click on the "Edit Rule" button to modify the selected rule.
9. In the "Source" field, specify the specific entities that you want to allow access from. This can be an individual IP address, a range of IP addresses, or a CIDR block.
10. Enter the specific IP address, IP range, or CIDR block in the "Source" field.
11. Optionally, you can also specify a description for the updated rule in the "Description" field.
12. Click on the "Save rules" button to apply the changes to the security group.
13. Review the updated inbound rules and ensure that the HTTP access is restricted to the specific entities you specified.
