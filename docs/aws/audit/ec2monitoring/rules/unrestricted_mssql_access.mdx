### More Info:

No security group should allow unrestricted inbound access to TCP port 1433 (MSSQL)

### Risk Level

Medium

### Address

Security

### Compliance Standards

SOC2, GDPR, HITRUST, NISTCSF, PCIDSS, FedRAMP

### Remediation

How to update your security groups inbound configuration to restrict MSSQL access to specific entities?

#### Using AWS Console

1. Log in to your AWS console and navigate to the EC2 dashboard.
2. Click on the "Security Groups" option from the left-hand menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unrestricted MsSQL Access Should Not Be Allowed" Policy.)
3. Select the security group that is associated with your MSSQL instance.
4. Click on the "Inbound Rules" tab.
5. Find the rule that allows MSSQL access and click the "Edit" button next to it.
6. In the "Source" field, select "Custom" and enter the IP address or range of IPs that should have access to the MSSQL instance.
7. Click "Save" to apply the new rule.
8. Test the new configuration to ensure that access to the MSSQL instance is restricted to the specified entities.
