### Remediation

How to secure EC2 AMIs from public access?

#### Using AWS Console

1. Log in to the AWS Management Console using your AWS account credentials.
2. Navigate to the EC2 service by selecting "EC2" from the services menu.
3. In the EC2 dashboard, click on "AMIs" in the left navigation pane.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "EC2 AMIs Should Not Be Public" Policy.)
4. Identify the AMIs that are set to public visibility.
5. Take note of the specific AMI identifier(s) that need remediation.
6. Select the checkbox next to the AMI(s) you want to remediate.
7. Above the list of AMIs, click on the "Actions" button.
8. In the dropdown menu, click on "Modify image permissions".
9. In the "Modify image permissions" dialog box, select the option for "Private".
10. Click on the "Save" button to update the permissions for the selected AMI(s).
11. Monitor the status of the permission update to ensure it is successfully applied to the AMIs.
12. Repeat these steps for each AMI that is set to public until all AMIs have been remediated.
