### More Info:

Your AWS servers should be using the latest generation of EC2 instances for price-performance improvements.

### Risk Level

Low

### Address

Cost optimization

### Compliance Standards

CBP

### Remediation

How to ensure EC2 latest generation is used?

#### Using AWS Console

1. Access the AWS Management Console and navigate to the Amazon EC2 service.
2. Click on "Instances" in the left navigation pane.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "EC2 Instances Should Use Latest Generation" Policy.)
3. Review the list of existing EC2 instances.
4. Identify any instances that are using older generation types.
5. Select each instance that is not using the latest generation.
6. Click on the "Actions" button and choose "Instance State" and then "Stop" to stop the selected instances.
7. Wait for the instances to stop.
8. Select the stopped instances again.
9. Click on the "Actions" button and choose "Instance Settings" and then "Change Instance Type".
10. In the instance type selection window, choose the latest generation instance type that matches your requirements.
11. Click on "Apply" to change the instance type.
12. Start the instances by selecting them and clicking on the "Actions" button, choosing "Instance State", and then "Start".
13. Monitor the instances to ensure they start up successfully with the latest generation instance type.
14. Repeat steps 5-13 for all instances that are not using the latest generation until all instances have been updated.

By following these steps, you can ensure that the latest generation of EC2 instances is used in your AWS environment. This ensures that you are taking advantage of the latest features, performance improvements, and cost optimizations available in the newer EC2 instance types.
