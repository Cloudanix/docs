### More Info:

There are EC2 instances scheduled for retirement and/or maintenance. Kindly take the necessary steps (reboot, restart or re-launch).

### Risk Level

Low

### Address

Security

### Compliance Standards

CBP

### Remediation

How to reboot, restart or relaunch and EC2 instance which is scheduled for retirement

#### Using AWS Console

1. Open the AWS Management Console and navigate to the Amazon EC2 service.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Scheduled Events for EC2 Instances" Policy.)
2. From the left navigation pane, click on "Instances" to view the list of EC2 instances.
3. Locate the EC2 instance that is scheduled for retirement and select it by clicking on the checkbox next to it.
4. From the "Actions" menu above the instance list, select "Instance State" and then click on "Stop".
5. In the confirmation dialog, click on "Stop" to initiate the instance stop process.
6. Wait for the instance to stop. You can monitor the status in the "Instance State" column.
7. Once the instance has stopped, select the instance again and click on the "Actions" menu.
8. From the "Actions" menu, select "Instance State" and then click on "Start".
9. In the confirmation dialog, click on "Start" to initiate the instance start process.
10. Wait for the instance to start. You can monitor the status in the "Instance State" column.
11. After the instance has started, you can access it using the same methods as before, such as connecting via SSH or accessing any applications running on the instance.
