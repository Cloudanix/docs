---
title: ECR Repository Tag Should Be Immutable
---

### More Info:

Ensures ECR repository image tags cannot be overwritten. ECR repositories should be configured to prevent overwriting of image tags to avoid potentially-malicious images from being deployed to live environments.

### Risk Level

High

### Address

Security, Operational Maturity

### Compliance Standards

CBP

### Remediation

How to ensure ECR repository tags are immutable

#### Using AWS Console

1. Open the Amazon ECR console.
2. Select the desired ECR repository from the list.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "ECR Repository Tag Should Be Immutable" Policy.)
3. In the repository details page, click on the "Images" tab.
4. Click on "Edit" button.
4. Scroll down to the "Tag mutability" radio button.
5. Ensure that the "Tag immutability" setting is enabled.
