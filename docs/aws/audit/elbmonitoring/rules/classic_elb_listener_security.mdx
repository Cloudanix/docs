### More Info:

Your Elastic Load Balancers (ELBs) listeners should not have insecure configurations. Only HTTPS or SSL should be used to encrypt the communication between the client and your load balancers.

### Risk Level

Medium

### Address

Security

### Compliance Standards

HIPAA, NIST, AWSWAF, GDPR

How to remove any insecure protocol definitions from ELB SSL negotiation settings?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the EC2 service.
2. From the EC2 dashboard, click on "Load Balancers" in the left navigation menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "ELBs Should Not Have Insecure Configurations" Policy.)
3. Select the ELB from which you want to remove insecure protocol definitions.
4. In the load balancer details page, go to the "Listeners" tab.
5. Select the listener that you want to modify.
6. Click on the "Edit" button next to the listener.
7. In the "Edit Listener" dialog, review the existing SSL negotiation configuration.
8. Identify any insecure protocol definitions, such as SSLv3 or TLSv1.0.
9. Remove the insecure protocol definitions by deselecting them or replacing them with more secure options. Ensure that only the desired secure protocols are selected.
10. Click on the "Save" button to save the changes to the listener.
