### More Info:

Elastic Load Balancers should be using the latest AWS predefined security policies.

### Risk Level

Medium

### Address

Security

### Compliance Standards

AWSWAF, PCIDSS

### Remediation

How to update ELB SSL negotiation configuration to use the latest AWS Predefined Security Policies?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the EC2 service.
2. From the EC2 dashboard, click on "Load Balancers" in the left navigation menu.
3. Select the ELB for which you want to update the SSL negotiation configuration.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "ELBs Must Use Latest AWS Security Policies" Policy.)
4. In the load balancer details page, go to the "Listeners" tab.
5. Under the "Listeners" section, you will see the list of configured listeners for the load balancer.
6. Select the listener that you want to update and click on the "Edit" button.
7. In the listener configuration dialog, locate the SSL/TLS section.
8. In the SSL/TLS section, you will find the "Security policy" field. Click on the dropdown menu to view the available AWS Predefined Security Policies.
9. Select the desired security policy from the list. AWS provides a set of predefined security policies with different encryption protocols and cipher suites.
10. Review the other configuration options in the dialog, such as the SSL certificate, port, and instance protocol, and make any necessary changes.
11. Click on the "Save" button to apply the changes to the listener configuration.
