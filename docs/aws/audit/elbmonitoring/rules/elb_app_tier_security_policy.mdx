### More Info:

Your app-tier Elastic Load Balancers (ELBs) listeners should be using the latest AWS security policy for their SSL negotiation configuration.

### Remediation

How to enable the latest predefined SSL security policy for app-tier ELBs?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the EC2 service.
2. From the EC2 dashboard, click on "Load Balancers" in the left navigation menu.
3. Select the ELB that represents your application-tier load balancer.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Latest AWS Security Policy for SSL Negotiations Should Be Used For App-Tier ELBs" Policy.)
4. In the ELB details page, go to the "Listeners" tab.
5. Identify the listener protocol and port for which you want to enable the latest SSL security policy and click on the "Change" link next to it.
6. In the "Configure Security Settings" dialog, select the desired security policy.
7. To enable the latest predefined SSL security policy, choose the policy with the highest version number. The policy names typically include "ELBSecurityPolicy-" followed by a date or version identifier.
8. Click on the "Save" button to apply the changes.
