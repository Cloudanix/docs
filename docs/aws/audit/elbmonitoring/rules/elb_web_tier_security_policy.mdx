### More Info:

Your web-tier Elastic Load Balancers (ELBs) listeners should be using the latest AWS security policy for their SSL negotiation configuration

### Remediation

How to enable the latest predefined SSL security policy for web-tier ELBs?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the EC2 service.
2. From the EC2 dashboard, click on "Load Balancers" in the left navigation menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Latest AWS Security Policy for SSL Negotiations Should Be Used For Web-Tier ELBs" Policy.)
3. Select the web-tier ELB for which you want to enable the latest SSL security policy.
4. In the load balancer details page, go to the "Listeners" tab.
5. Select the listener for which you want to enable the latest SSL security policy.
6. Click on the "Edit" button next to the listener.
7. In the "Edit Listener" dialog, select "HTTPS" as the protocol for the listener.
8. Specify the desired port for the listener, typically port 443 for HTTPS traffic.
9. In the "SSL certificate" section, select the SSL certificate that you have already uploaded to AWS Certificate Manager (ACM) or IAM.
   - If you have already uploaded the SSL certificate to ACM, you can select it from the dropdown menu.   
   - If you have uploaded the SSL certificate to IAM, you can choose "Choose a certificate from ACM or IAM" and then select the appropriate certificate.
10. In the "Security policy" section, select the option "Predefined security policy".
11. Choose the latest predefined SSL security policy from the dropdown menu. The available options include AWS predefined security policies such as ELBSecurityPolicy-2016-08, ELBSecurityPolicy-TLS-1-1-2017-01, ELBSecurityPolicy-TLS-1-2-2017-01, etc.
12. Click on the "Save" button to save the changes to the listener.
