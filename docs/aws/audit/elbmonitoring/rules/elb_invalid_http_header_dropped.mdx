---
title: ELBs Should Drop Invalid HTTP Header
---

### More Info:

Invalid HTTP Headers in ELB should be dropped.

### Risk Level

High

### Address

Security

### Compliance Standards

CBP

### Remediation

How to drop an invalid HTTP header?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the EC2 service.
2. From the EC2 dashboard, click on "Load Balancers" in the left navigation menu.
3. Select the ELB for which you want to configure the security policy.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "ELBs Should Drop Invalid HTTP Header" Policy.)
4. In the ELB details page, go to the "Listeners" tab.
5. Identify the listener protocol (HTTP or HTTPS) for which you want to configure the security policy and click on the "Change" link next to it.
6. In the "Configure Security Settings" dialog, select the desired security policy.
7. To drop invalid HTTP headers, choose a security policy that includes the "ELBSecurityPolicy-2016-08" or a newer version. These security policies are designed to drop invalid or unsafe HTTP headers.
8. Click on the "Save" button to apply the changes.
