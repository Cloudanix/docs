### More Info:

Your Amazon ALBs should be using the latest predefined security policy for their SSL negotiation configuration in order to follow security best practices and protect their front-end connections against SSL/TLS vulnerabilities.

### Risk Level

Medium

### Address

Security

### Compliance Standards

AWSWAF, HIPAA, GDPR, NIST

### Remediation

How to check if ALBs have latest SSL/TLS configuration?

#### Using AWS Console

1. Open the AWS Management Console and navigate to the EC2 service.
2. From the EC2 dashboard, click on "Load Balancers" in the left navigation menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "ALBs Should Have Latest SSL/TLS Configurations" Policy.)
3. You will see a list of load balancers in your account. Look for the ALBs for which you want to check the SSL/TLS configuration.
4. Select the ALB that you want to check.
5. In the ALB details page, click on the "Listeners" tab.
6. You will see a list of listeners associated with the ALB. Each listener represents a specific protocol and port configuration.
7. Review the SSL/TLS configuration for each listener. Specifically, check the SSL policy being used.
   - Click on the listener to expand its details.
   - Look for the "SSL policy" field, which indicates the SSL policy currently configured for the listener.
   - Compare the SSL policy with the latest SSL/TLS policies provided by AWS.
