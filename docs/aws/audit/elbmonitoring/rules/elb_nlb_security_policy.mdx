### More Info:

Your Amazon Network Load Balancers (NLBs) should be using the latest recommended predefined security policy for TLS negotiation configuration in order to protect their front-end connections against TLS vulnerabilities and meet security requirements

### Risk Level

Medium

### Address

Security

### Compliance Standards

HITRUST, AWSWAF, HIPAA, GDPR, NIST

### Remediation

How to update Amazon Network Load Balancers (NLBs) listeners configuration?

#### Using AWS Console

1. Open the Amazon EC2 Management Console and navigate to the "Load Balancers" section.
2. Select your desired NLB from the list of load balancers.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "NLBs Should Have Latest SSL/TLS Configurations" Policy.)
3. In the "Listeners" tab, you will see the existing listeners configured for the NLB.
4. To update a listener, click on the "Edit" button next to the listener's configuration.
5. In the listener configuration, you can modify settings such as the protocol, port, and SSL certificate.
6. Make the necessary changes to the listener configuration and click "Save" to apply the updates.
7. Depending on the changes you made, there may be a brief interruption in traffic as the NLB applies the new configuration.
8. Verify that the listener updates have been applied correctly by testing the desired functionality or reviewing the listener configuration.
