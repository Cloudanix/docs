---
title: Enable Encryption For API Cache
---

### More Info:

Ensure that your Amazon API Gateway REST APIs are configured to encrypt API cached responses in order to protect data while in transit (as it travels to and from Amazon API Gateway).

### Risk Level

High

### Address

Security

### Compliance Standards

CBP,SOC2,NISTCSF,PCIDSS

### Remediation

How to enable encryption for API Cache

#### Using AWS Console

1. Log in to the AWS Management Console using your AWS account credentials.
2. Navigate to the API Gateway service by selecting "API Gateway" from the services menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Enable Encryption For API Cache" Policy.)
3. In the API Gateway dashboard, select the API for which you want to enable encryption for the cache.
4. Click on the "Stages" tab in the left navigation pane.
5. Select the desired stage for which you want to enable cache encryption.
6. In the stage details page, locate the "Cache" section.
7. Click on the "Edit" button next to the cache settings.
8. In the "Edit cache settings" dialog box, enable the "Enable encryption at rest" option.
9. Select the AWS Key Management Service (KMS) customer managed key (CMK) or AWS Managed Key (default) to use for encryption.
10. Click on the "Save" button to apply the encryption settings to the cache.
11. The API cache will now be encrypted at rest using the specified encryption key.
