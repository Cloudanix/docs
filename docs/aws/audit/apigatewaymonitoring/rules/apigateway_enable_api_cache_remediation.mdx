   ```
   aws apigateway create-cache-cluster --cache-cluster-name <cluster-name> --cache-node-type <node-type> --region <region>
   ```

   Replace `<cluster-name>` with a name for your cache cluster, `<node-type>` with the type of nodes you want to use (e.g. `cache.t2.micro`), and `<region>` with the AWS region you want to create the cache in.

3. Once the cache cluster is created, you need to enable caching for your API. Run the following command to get the ID of your API:

   ```
   aws apigateway get-rest-apis --query "items[?name=='<api-name>'].id" --output text
   ```

   Replace `<api-name>` with the name of your API.

4. Use the API ID to enable caching for your API by running the following command:

   ```
   aws apigateway update-rest-api --rest-api-id <api-id> --patch-operations '[{"op":"replace","path":"/endpointConfiguration/types/EDGE/caching/enabled","value":"true"}]'
   ```

   Replace `<api-id>` with the ID of your API.

5. Finally, deploy your API to make the caching changes take effect by running the following command:

   ```
   aws apigateway create-deployment --rest-api-id <api-id> --stage-name <stage-name>
   ```

   Replace `<api-id>` with the ID of your API and `<stage-name>` with the name of the deployment stage you want to use.
