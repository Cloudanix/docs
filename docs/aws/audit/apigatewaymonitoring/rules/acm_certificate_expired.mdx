### More Info:

Ensure that all the expired Secure Sockets Layer/Transport Layer Security (SSL/TLS) certificates managed by AWS Certificate Manager are removed in order to adhere to Amazon Security Best Practices. Certificate Manager is the AWS service that lets you easily provision, manage, and deploy SSL/TLS certificates for use with other Amazon services such as Elastic Load Balancing and CloudFront.

### Risk Level

High

### Address

Security

### Remediation

How to remediate if API gateway ACM certificate is expired?

#### Using AWS Console

1. Log in to the AWS Management Console using your AWS account credentials.
2. Navigate to the API Gateway service by selecting "API Gateway" from the services menu.
    (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "ACM Certificate Expired" Policy.)
3. In the API Gateway dashboard, select the API Gateway instance where the ACM certificate is expired.
4. Click on the "Custom Domain Names" tab in the left navigation pane.
5. Select the custom domain name associated with the expired ACM certificate.
6. In the domain details page, locate the "Certificate" section.
7. Click on the "Edit" button next to the certificate.
8. In the "Edit Custom Domain Name" dialog box, select the "Request or Import a Certificate with ACM" option.
9. Click on the "Request a new certificate" button.
10. Follow the on-screen instructions to request a new ACM certificate for your domain.
11. Make sure to validate the ownership of the domain for which you are requesting the certificate.
12. Once the new ACM certificate is issued and validated, return to the "Edit Custom Domain Name" dialog box.
13. Select the newly issued certificate from the dropdown menu.
14. Click on the "Save" button to apply the new certificate to the custom domain name.
15. The expired ACM certificate will be replaced with the new certificate for the API Gateway instance.
16. Test the API Gateway to ensure that the new certificate is properly configured and working.
