### Remediation

#### Using Console

To remediate the misconfiguration of AWS ACM Certificates Renewal Under 30 Days, you can follow the below steps in the AWS console:

1. Login to the AWS Console and navigate to the AWS Certificate Manager (ACM) service.

2. Select the certificate that is expiring soon and needs to be renewed.

3. Click on the "Renew" button to renew the certificate.

4. Follow the on-screen instructions to complete the renewal process.

5. Once the certificate is renewed, update the certificate on all the resources that are using it, such as Load Balancers, CloudFront distributions, and API Gateway.

6. Verify that the updated certificate is being used by all the resources.

7. Set up a reminder or notification to renew the certificate before it expires again.

By following the above steps, you can remediate the misconfiguration of AWS ACM Certificates Renewal Under 30 Days and ensure that your resources are using valid certificates.

#### Using CLI

The following steps can be taken to remediate the AWS ACM Certificates Renewal Under 30 Days issue using the AWS CLI:

1. Open the AWS CLI on your local machine.

2. Use the following command to list all the certificates that are expiring in the next 30 days:

   ```
   aws acm list-certificates --query "CertificateSummaryList[?ExpirationDate<='$(date -d '+30 days' --utc +%Y-%m-%dT%H:%M:%SZ')']"
   ```

   This command will return a list of certificates that are expiring in the next 30 days.

3. Identify the certificate that needs to be renewed.

4. Use the following command to request a new certificate:

   ```
   aws acm request-certificate --domain-name example.com --validation-method DNS
   ```

   Replace "example.com" with the domain name for which you want to request a new certificate.

5. Follow the instructions provided by AWS to complete the certificate validation process. This usually involves creating a DNS record.

6. Once the validation process is complete, use the following command to update the certificate:

   ```
   aws acm import-certificate --certificate file://certificate.pem --certificate-chain file://certificate_chain.pem --private-key file://private_key.pem
   ```

   Replace "certificate.pem", "certificate_chain.pem", and "private_key.pem" with the paths to the new certificate, certificate chain, and private key files.

7. Verify that the new certificate is in use by checking the certificate details in the AWS Management Console or by running the following command:

   ```
   aws acm describe-certificate --certificate-arn arn:aws:acm:us-west-2:123456789012:certificate/12345678-1234-1234-1234-123456789012
   ```

   Replace "arn:aws:acm:us-west-2:123456789012:certificate/12345678-1234-1234-1234-123456789012" with the ARN of the new certificate.
