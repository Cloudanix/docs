### More Info:

Ensure that response caching is enabled for your Amazon API Gateway REST APIs in order to enhance API responsiveness and decrease latency.

### Remediation

How to enable API Cache for API Gateway?

#### Using AWS Console

To enable API caching for API Gateway in AWS, you can follow these steps:

1. Open the AWS Management Console: https://console.aws.amazon.com/

2. Navigate to the API Gateway service by selecting it from the list of available services.

3. In the API Gateway console, locate the API you want to enable caching for and click on its name to open the API settings.

4. In the API's settings, select the "Stages" tab. This will show a list of stages for the selected API.

5. Select the stage for which you want to enable caching by clicking on its name.

6. In the stage settings, scroll down to the "Cache" section and click on the "Enable API Cache" button.

7. Configure the cache settings as per your requirements. You can specify the cache capacity, TTL (time to live) for cached responses, and other cache-related settings.

8. Once you have configured the cache settings, click on the "Save Changes" button to enable caching for the API stage.

9. The API caching will be enabled for the selected stage of your API Gateway. Cached responses will be served from the cache based on the configured TTL and cache key settings.

Enabling API caching can help improve the performance and reduce the load on your backend services by serving cached responses to client requests. Make sure to configure the cache settings appropriately based on your API's behavior and requirements.
