   ```
   aws es describe-elasticsearch-domain --domain-name <your-domain-name>
   ```
   Replace `<your-domain-name>` with the name of your Elasticsearch domain. This command will return a JSON output with the configuration details of the specified Elasticsearch domain.

4. Check for VPC options: In the JSON output, look for the "VPCOptions" field. If the "VPCOptions" field is not present or if the "VPCOptions" field does not contain "Endpoint" and "SecurityGroupIds", then the Elasticsearch domain is not launched with a private VPC endpoint. 

   Here is an example of what the "VPCOptions" field should look like for a domain with a private VPC endpoint:

   ```
   "VPCOptions": {
       "SubnetIds": [
           "subnet-0abcd1234efgh5678"
       ],
       "VPCId": "vpc-0abcd1234efgh5678",
       "SecurityGroupIds": [
           "sg-0abcd1234efgh5678"
       ],
       "Endpoint": "vpc-<your-domain-name>-abcd1234efgh5678.es.amazonaws.com"
   }
   ```
   If the "VPCOptions" field is missing or does not contain "Endpoint" and "SecurityGroupIds", then the Elasticsearch domain is not launched with a private VPC endpoint.

   ```python
   import boto3

   def check_es_vpc_endpoints():
       es = boto3.client('es')
       domains = es.list_domain_names()
       for domain in domains['DomainNames']:
           domain_info = es.describe_elasticsearch_domain(DomainName=domain['DomainName'])
           if 'VPCOptions' in domain_info['DomainStatus']:
               if 'VPCId' in domain_info['DomainStatus']['VPCOptions']:
                   print(f"Domain {domain['DomainName']} is launched with a private VPC endpoint.")
               else:
                   print(f"Domain {domain['DomainName']} is not launched with a private VPC endpoint.")
           else:
               print(f"Domain {domain['DomainName']} is not launched with a private VPC endpoint.")

       check_es_vpc_endpoints()
   ```
