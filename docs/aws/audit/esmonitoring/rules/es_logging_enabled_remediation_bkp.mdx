### Remediation

How to ensure logging is enabled for ElasticSearch?

#### Using AWS Console

1. Sign in to the AWS Management Console.
2. Navigate to the Amazon Elasticsearch service.
   (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "ElasticSearch Should Have Logging Enabled" Policy.)
3. Select the Elasticsearch domain for which you want to enable logging.
4. In the domain details page, click on the "Configure" button in the "Monitoring" section.
5. Under the "Logging" section, click on the "Edit" button.
6. Enable the "Enable slow search logs" option to capture slow search queries in the logs.
7. Choose the desired log publishing destination. You can select either Amazon CloudWatch Logs or an existing Amazon S3 bucket.
8. If you choose to publish logs to CloudWatch Logs, select the appropriate CloudWatch log group or create a new one.
9. If you choose to publish logs to an existing S3 bucket, select the bucket from the dropdown list.
10. Click on the "Save changes" button to apply the configuration.
