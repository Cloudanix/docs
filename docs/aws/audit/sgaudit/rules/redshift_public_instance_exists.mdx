### More Info:

Redshift clusters should not be launched into the public cloud. Unless there is a specific business requirement, Redshift clusters should not have a public endpoint and should be accessed from within a VPC only.

### Risk Level

High

### Address

Security

### Compliance Standards

HIPAA

### Remediation

How to remediate if Redshift clusters are public

#### Using AWS Console

If you have identified that your Amazon Redshift clusters are publicly accessible, you can remediate the issue by following these steps:

1. Open the AWS Management Console and navigate to the Amazon Redshift service.

2. Select the Redshift cluster that is publicly accessible.

3. In the cluster details page, click on the "Properties" tab.

4. Locate the "Publicly Accessible" setting and change it to "No" to disable public access.

5. Review the other configuration settings and make any necessary changes.

6. Click on the "Modify" button to save the changes.

7. In the confirmation dialog, review the changes and click on the "Modify Cluster" button.

Once the modification is complete, your Redshift cluster will no longer be publicly accessible. It will only be accessible from within your VPC or through authorized connections.

It is recommended to regularly review and monitor the accessibility settings of your Redshift clusters to ensure that they align with your security requirements and best practices.
