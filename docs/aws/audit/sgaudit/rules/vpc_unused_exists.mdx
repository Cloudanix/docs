---
title: Unused Virtual Private Gateways Should Be Removed
---

### More Info:

Unused Amazon Virtual Private Gateways should be removed in order to adhere to best practices and to avoid reaching the service limit.

### Risk Level

Low

### Address

Operational Maturity

### Compliance Standards

CBP, APRA, MAS, NIST

### Remediation

How to remove unused VPC?

#### Using AWS Console

1. Identify Unused VPC:
     (In the Cloudanix Console, navigate to "Misconfig" page and look for Affected Assets for "Unused Virtual Private Gateways Should Be Removed" Policy.)
   - Open the AWS Management Console and navigate to the Amazon VPC service.
   - Review the list of VPCs and identify the VPC that you want to remove.
2. Check Resource Dependencies:
   - Check if there are any EC2 instances, RDS databases, Elastic Load Balancers, or other resources associated with the VPC.
   - Review each subnet within the VPC and verify if there are any resources deployed within them.
   - Ensure that all resources associated with the VPC are either deleted or moved to a different VPC.
3. Detach Internet Gateways (if applicable):
   - If the VPC has an Internet Gateway attached, ensure that it is detached from the VPC.
   - Open the Amazon VPC console, navigate to the "Internet Gateways" section, select the Internet Gateway associated with the VPC, and click on the "Actions" button.
   - Choose "Detach from VPC" and confirm the detachment.
4. Delete Route Tables:
   - Open the Amazon VPC console, navigate to the "Route Tables" section, and identify the route table associated with the VPC.
   - Select the route table and click on the "Actions" button.
   - Choose "Delete route table" and confirm the deletion.
5. Delete Subnets:
   - Open the Amazon VPC console, navigate to the "Subnets" section, and identify the subnets within the VPC.
   - Select each subnet and click on the "Actions" button.
   - Choose "Delete subnet" and confirm the deletion.
6. Remove Security Groups:
   - Open the Amazon VPC console, navigate to the "Security Groups" section, and identify the security groups associated with the VPC.
   - Select each security group and click on the "Actions" button.
   - Choose "Delete security group" and confirm the deletion.
7. Delete Network Interfaces (if applicable):
   - Open the Amazon EC2 console and navigate to the "Network Interfaces" section.
   - Identify any network interfaces associated with the VPC and delete them.
   - Ensure that there are no active network interfaces associated with the VPC.
8. Remove VPC Peering Connections (if applicable):
   - If the VPC has any peering connections with other VPCs, remove the peering connections.
   - Open the Amazon VPC console, navigate to the "Peering Connections" section, and select the peering connections associated with the VPC.
   - Click on the "Actions" button and choose "Delete peering connection" to remove the connections.
9. Delete VPC:
   - Once all the resources associated with the VPC have been removed, open the Amazon VPC console, select the VPC, and click on the "Actions" button.
   - Choose "Delete VPC" and confirm the deletion.
10. Verify Removal:
    - After confirming the deletion, monitor the deletion progress in the Amazon VPC console.
    - Ensure that the VPC is completely removed and no longer appears in the VPC list.
